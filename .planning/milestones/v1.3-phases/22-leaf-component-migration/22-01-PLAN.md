  <action>
For each of the 13 component files listed, apply the following mechanical transform:

1. Add `<svelte:options runes />` as the first line of the file (before the HTML comment block).

2. Remove the `type $$Props = ComponentProps;` line.

3. Replace all `export let` declarations with a single destructured `$props()` call:
   ```svelte
   <!-- BEFORE -->
   export let text: $$Props['text'];
   export let variant: $$Props['variant'] = 'normal';

   <!-- AFTER -->
   let { text, variant = 'normal', ...restProps }: ComponentProps = $props();
   ```
   Use the actual Props type name from the `.type.ts` import (e.g., `ErrorMessageProps`, `LoadingProps`, `WarningProps`, etc.).

4. Replace all `$$restProps` references with `restProps`:
   - `concatClass($$restProps, ...)` -> `concatClass(restProps, ...)`
   - `concatProps($$restProps, ...)` -> `concatProps(restProps, ...)`
   - `{...$$restProps}` -> `{...restProps}`

5. If the component has `$:` reactive statements, convert them:
   - `$: x = expression` -> `let x = $derived(expression)`
   - `$: { ... multi-statement ... }` -> `let x = $derived.by(() => { ... return result; })`
   - Side-effect-only `$:` blocks -> `$effect(() => { ... })`

6. If the component has `on:event={handler}` on HTML elements (NOT on child Svelte components), convert to `onevent={handler}`:
   - `on:click={handler}` -> `onclick={handler}`
   - `on:keydown={handler}` -> `onkeydown={handler}`
   - etc.

7. If the component has a default `<slot />`, leave it as-is (slots work in runes mode, conversion is Phase 23).

Components with special attention:
- **Term.svelte**: Has a default `<slot />` -- keep it, just migrate props.
- **HeadingGroup.svelte**: Has a default `<slot />` -- keep it, just migrate props.
- **PreHeading.svelte**: Has a default `<slot />` -- keep it, just migrate props.
- **Warning.svelte**: Has a default `<slot />` -- keep it, just migrate props.
