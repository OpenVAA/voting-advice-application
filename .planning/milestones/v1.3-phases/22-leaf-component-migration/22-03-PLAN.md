Video.svelte (884 lines) is the most complex leaf component. Apply the full migration with these specific transformations:

**1. Replace `<svelte:options accessors />` with `<svelte:options runes />`**
In runes mode, the `accessors` option is ignored -- all exports are automatically accessible via `bind:this`. Remove `accessors` and add `runes`.

**2. Convert props with $bindable():**
Search for `bind:` usage of Video props across the codebase:
```bash
grep -rn 'bind:.*Video\|<Video.*bind:' apps/frontend/src/ --include="*.svelte"
```
Props that consumers bind to need `$bindable()`. Based on research, these include:
- `atEnd` (bound by layout context) -> `atEnd = $bindable(false)`
- `mode` (bound by layout context) -> `mode = $bindable('video')`
- Any other props confirmed by grep

The full props destructuring:
```svelte
let {
  src,
  title,
  poster,
  captions,
  transcript,
  // ... other props with defaults ...
  atEnd = $bindable(false),
  mode = $bindable('video'),
  ...restProps
}: VideoProps = $props();
```

**3. Convert 17+ reactive statements:**
Classify each `$:` statement in Video.svelte:

- **Pure derivations** (no side effects, just computing a value):
  `$: effectiveTitle = ...` -> `let effectiveTitle = $derived(...)`

- **Props that are set internally AND bound externally** (Pitfall 7 from research):
  `$: atEnd = isAtEnd(currentTime)` -> `$effect(() => { atEnd = isAtEnd(currentTime); })`
  This uses `$effect` because it MUTATES a `$bindable()` prop, which is a side effect.

- **Multi-statement blocks**:
  `$: { classes = ...; ... }` -> `let classes = $derived.by(() => { ... return c; })`

- **Side effects** (DOM manipulation, dispatching, etc.):
  `$: { if (condition) doSomething(); }` -> `$effect(() => { if (condition) doSomething(); })`

**4. Replace event modifiers (COMP-09):**
All event modifiers are in Video.svelte:

**|once replacement:**
Create a reusable `once()` wrapper in the script block:
```typescript
function once<T extends (...args: any[]) => any>(fn: T) {
  let called = false;
  return function (this: any, ...args: Parameters<T>) {
    if (called) return;
    called = true;
    return fn.apply(this, args);
  } as T;
}
```

Then replace each `on:click|once={tryUnmute}` with `onclick={once(tryUnmute)}`.

IMPORTANT: Since `tryUnmute` is called from MULTIPLE buttons each with `|once`, each button needs its own `once` wrapper instance. Pre-create them in the script block:
```typescript
const tryUnmuteOnce1 = once(tryUnmute);
const tryUnmuteOnce2 = once(tryUnmute);
// ... for each button
```
Or better: evaluate whether `tryUnmute` is already idempotent (check its implementation). If it already handles the "called once" semantic internally, the `once` wrapper may be unnecessary and you can just use `onclick={tryUnmute}`.

**|capture replacement:**
Replace `on:click|capture={handler}` with `onclickcapture={handler}`:
```svelte
<!-- BEFORE -->
on:click|capture={() => screenJump(-1)}

<!-- AFTER -->
onclickcapture={() => screenJump(-1)}
```

**Combined examples:**
```svelte
<!-- BEFORE -->
on:click|once={tryUnmute}       -> onclick={once(tryUnmute)}  OR onclick={tryUnmute} if idempotent
on:click|capture={handler}      -> onclickcapture={handler}
```

**5. Convert all other on:event handlers to onevent:**
- `on:click={handler}` -> `onclick={handler}`
- `on:keydown={handler}` -> `onkeydown={handler}`

- etc.

Also convert `<svelte:document on:keydown=...>` -> `<svelte:document onkeydown=...>` and `<svelte:window on:resize=...>` -> `<svelte:window onresize=...>`.

`bind:currentTime`, `bind:duration`, `bind:paused`, `bind:volume`, `bind:muted` etc. on HTML `<video>` elements stay as-is -- these are HTML element bindings, not component prop bindings, and work unchanged in runes mode.
