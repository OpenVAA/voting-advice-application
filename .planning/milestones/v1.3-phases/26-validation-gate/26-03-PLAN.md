## Entity Detail Navigation Pattern (pushState + Drawer)

The results page intercepts entity card link clicks via `beforeNavigate`:

```typescript
// apps/frontend/src/routes/(voters)/(located)/results/+page.svelte lines 163-175
beforeNavigate(({ cancel, to }) => {
  if (to?.route.id !== ROUTE.ResultEntity) return;
  const { entityType, entityId, nominationId } = parseParams(to);
  if (!entityType || !entityId || !nominationId || Array.isArray(nominationId)) return;
  pushState(to.url, {
    resultsShowEntity: { entityType, entityId, nominationId }
  });
  cancel();
});
```

When an entity card `<a href="/results/candidate/[id]">` is clicked:
1. `beforeNavigate` fires, calls `pushState(url, state)` then `cancel()`
2. The browser URL changes to `/results/candidate/[id]` (pushState)
3. NO actual SvelteKit route navigation occurs (cancel prevents it)
4. `page.state.resultsShowEntity` becomes truthy
5. The `EntityDetailsDrawer` renders inside a `<dialog>` element (via ModalContainer)
6. `EntityDetails` (with `data-testid="entity-details"`) renders inside the Drawer

THEREFORE: Tests must NOT expect page-level navigation. They must:
- Use `waitForURL` to detect the pushState URL change
- Locate `entity-details` inside the dialog/drawer overlay (NOT as a page element)
- Close the drawer to go back (NOT use page.goBack())

## Results Page Reactivity Architecture

```
activeElectionId (set sync for single election at line 111)
  -> $effect (line 114): sets entityTabs, activeEntityType, activeElection
  -> $effect (line 127): sets activeMatches from $matches[activeElectionId][activeEntityType]
```

Both $effects run as microtasks (async). When `handleEntityTabChange` sets
`activeEntityType`, the second $effect updates `activeMatches` in the next microtask.
During this gap, `activeMatches` may briefly be stale or the {#key activeMatches} block
may be re-rendering.

The voter-results-list (data-testid) renders at:
```svelte
{#if activeMatches}
  <div data-testid={activeEntityType === 'candidate' ? 'voter-results-candidate-section' : ...}>
    {#key activeMatches}
      <EntityList ... data-testid="voter-results-list" />
    {/key}
  </div>
{/if}
```
