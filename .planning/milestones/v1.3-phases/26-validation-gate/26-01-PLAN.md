<!-- Key patterns the executor needs for fixing TypeScript narrowing errors -->

All 11 errors follow the same pattern: `$state<T>()` without initializer creates `T | undefined`.
When used without null guard, svelte-check reports "'X' is possibly 'undefined'".

Fix strategies:
1. Guard clause in functions: `if (!variable) return;`
2. Non-null assertion in template (when parent {#if} guard exists): `variable!`
3. Optional chaining for property access: `variable?.method()`

The variables involved:
- `question` declared as `let question = $state<AnyQuestionVariant>();` -- undefined until $effect runs
- `questionBlock` declared as `let questionBlock = $state<...>();` -- undefined until $effect runs
- `category` declared as `let category = $state<QuestionCategory>();` -- undefined until $effect runs
- `activeElection` declared as `let activeElection = $state<Election>();` -- undefined until $effect runs

All are inside components that have `{#if variable}` guards in the template, so template usage
is protected. The errors are in:
1. `onMount` callbacks that run after $effect has populated values
2. Template expressions inside the `{#if}` guard where TypeScript doesn't narrow through Svelte `{#if}`
3. Callback closures passed as props (where TypeScript can't prove the value is defined at call time)
</interfaces>
