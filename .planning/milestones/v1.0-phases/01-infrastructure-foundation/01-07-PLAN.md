**Dynamic components that need testId props or direct attributes:**

Read each file listed above and:

1. **Navigation.svelte / NavItem.svelte**: Add `data-testid` on the nav container and individual nav links. NavItem likely wraps an `<a>` or `<button>` -- place testId on the interactive element. Use pattern: `nav-menu`, `nav-menu-item` (or accept a `testId` prop to make them dynamic).

2. **EntityCard.svelte / EntityCardAction.svelte**: Add `data-testid` on the card container and the action button/link. These render result cards. Use `data-testid` directly or accept a `testId` prop.

3. **EntityDetails.svelte**: Add `data-testid` on the detail container.

4. **EntityList.svelte / EntityListControls.svelte**: Add `data-testid` on the list container and filter/sort controls.

5. **Feedback.svelte**: Add `data-testid` on the feedback form container, input, and submit button.

6. **SurveyButton.svelte / SurveyBanner.svelte**: Add `data-testid` on the survey trigger button and banner.

7. **ConstituencySelector.svelte**: Add `data-testid` on the selector container and individual constituency options.

8. **ElectionSelector.svelte**: Add `data-testid` on the selector container and individual election options.

9. **Tabs.svelte**: Add `data-testid` on each tab button. If tabs are rendered dynamically, accept a `testId` prefix prop and append tab index/name.

10. **QuestionChoices.svelte**: Add `data-testid` on each choice option (Likert scale buttons, etc.). This is critical for voter question answering. Use pattern like `voter-questions-option` with index or value suffix.

11. **QuestionActions.svelte**: Add `data-testid` on next/previous/skip action buttons.

12. **OpinionQuestionInput.svelte**: Add `data-testid` on the input wrapper.

**Rules:**

- For components that accept `$$restProps` and spread on the right element: just verify, no change needed
- For components that need explicit testId support: add an optional `testId` prop that gets placed on the interactive element
- For dynamically rendered lists: use the `testId` as a base with index/key suffix
- Do NOT break any existing functionality -- only add attributes
- Check if the component already has `$$restProps` spread -- if it does on the right element, it already supports `data-testid` as a forwarded prop
