1. Install `eslint-plugin-playwright` as a dev dependency:
   ```bash
   yarn add -D eslint-plugin-playwright@^2.9.0
   ```
   Install at the root workspace level (the `tests/` directory doesn't have its own package.json — the root package.json has the ESLint dependency).

2. Rewrite `tests/eslint.config.mjs`:

   ```javascript
   import { default as sharedConfig } from '@openvaa/shared-config/eslint';
   import playwright from 'eslint-plugin-playwright';

   export default [
     ...sharedConfig,
     {
       ignores: ['playwright*']
     },
     {
       // Apply Playwright rules to all test files
       files: ['tests/**/*.ts'],
       ...playwright.configs['flat/recommended'],
       rules: {
         ...playwright.configs['flat/recommended'].rules,

         // === Error rules (hard enforcement) ===
         // Enforce getByTestId over raw CSS/text selectors
         'playwright/no-raw-locators': 'error',
         // No waitForTimeout — use waitFor conditions instead
         'playwright/no-wait-for-timeout': 'error',
         // Must await Playwright async methods
         'playwright/missing-playwright-await': 'error',
         // No test.only in committed code
         'playwright/no-focused-test': 'error',
         // Prefer web-first assertions (toBeVisible over manual checks)
         'playwright/prefer-web-first-assertions': 'error',

         // === Warning rules (soft enforcement) ===
         // No page.pause() in committed code
         'playwright/no-page-pause': 'warn',
         // No test.skip — prefer conditional skip
         'playwright/no-skipped-test': 'warn',

         // === Disable shared config rules that conflict with test patterns ===
         // Tests use arrow functions in test() callbacks
         'func-style': 'off'
       }
     }
   ];
   ```

3. Verify the config works by running ESLint on the existing test files:

   ```bash
   cd tests && npx eslint tests/ --max-warnings=0 2>&1 | head -30
   ```

   Expect some warnings/errors on existing spec files (they use text-based selectors, which is exactly what we want to flag). The existing files will be migrated in Phase 2.

4. If there are unexpected errors from the shared config conflicting with the Playwright plugin, add additional rule overrides in the `rules` object. Common conflicts:

   - `func-style` (already disabled above)
   - Import ordering rules that conflict with test setup patterns
   - TypeScript strict rules that conflict with Playwright's assertion patterns

5. Do NOT fix existing test files — they are legacy and will be rewritten in Phase 2. The ESLint config should flag them as having issues, which is correct behavior.
