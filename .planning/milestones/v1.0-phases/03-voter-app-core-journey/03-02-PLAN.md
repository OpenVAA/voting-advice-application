Create `voter-journey.spec.ts` in `tests/tests/specs/voter/`. This spec uses serial mode (`test.describe.configure({ mode: 'serial' })`) because tests represent sequential flow steps and share page state.

Structure the spec as a serial describe block with these tests:

**Test: "should load home page and display start button" (VOTE-01)**
- Navigate to `buildRoute({ route: 'Home', locale: 'en' })`
- Assert `voterHomePage.startButton` is visible
- Click start button
- Verify navigation away from home page

**Test: "should auto-imply election and constituency" (VOTE-02, VOTE-03)**
- After clicking start on home, auto-implication should skip both election and constituency selection
- Assert the election selection page is NOT shown: `page.getByTestId(testIds.voter.elections.list)` should NOT be visible
- Assert the constituency selection page is NOT shown: `page.getByTestId(testIds.voter.constituencies.list)` should NOT be visible
- Assert that the page has navigated directly to intro (URL contains `/intro`)
- This verifies VOTE-02 (single election auto-implied) and VOTE-03 (single constituency auto-implied)

**Test: "should show questions intro page with start button" (VOTE-04)**
- Assert we're on the intro page: URL contains `/intro`
- Assert `voterIntroPage.startButton` is visible
- Assert category intro element is NOT visible: `page.getByTestId(testIds.voter.questions.categoryIntro)` should not be visible (category intros disabled via app settings)
- Click start button
- Verify navigation to first question (URL contains `/questions/`)

**Test: "should answer all 8 Likert questions with navigation" (VOTE-06)**
- Should now be on the first question page
- The CONTEXT says: "Answer all 8 Likert questions to exercise full question navigation (next, previous, skip, last-question behavior)." Embed navigation testing within the flow:

1. Answer question 1 (select "Fully agree" index 4 via `voterQuestionsPage.selectAnswer(4)`), wait for auto-advance to Q2
2. Go back to Q1 (click `voterQuestionsPage.goPrevious()`), verify Q1 is displayed (URL changes)
3. Re-answer Q1, auto-advance to Q2
4. Answer Q2, auto-advance to Q3
5. Skip Q3 (click `voterQuestionsPage.goNext()` without answering -- shows "Skip" label), advance to Q4
6. Go back to Q3, answer Q3, advance to Q4
7. Answer Q4-Q7 normally (select answer, wait for auto-advance each time)
8. Answer Q8 (last question), then click the next/results button (`voterQuestionsPage.goNext()`)
9. Verify results page: assert URL contains `/results` and `page.getByTestId(testIds.voter.results.list)` is visible

NOTE on auto-advance: After `selectAnswer()`, the app auto-advances after a brief delay via `setTimeout(handleJump, DELAY)`. After selecting an answer, wait for the URL to change to the next question. Do NOT click next explicitly for non-last questions -- that would cause double navigation. For the last question (Q8), the next button label changes to "Results" after answering; click it to navigate to results.

For skip: click `goNext()` WITHOUT selecting an answer first. The button shows "Skip" when no answer is selected.
For previous: click `goPrevious()` at any time.
