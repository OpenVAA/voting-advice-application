From frontend constituency selection page behavior:
- ConstituencySelector renders radio buttons/dropdowns per constituency group
- When `useSingleGroup` (startFromConstituencyGroup mode), only that group's constituencies are shown
- The `selectionComplete` boolean bound from ConstituencySelector enables the Continue button
- After constituency selection + continue, redirects to questions (normal flow) or elections (startFromConstituencyGroup flow)

From frontend election selection page with startFromConstituencyGroup:
- Filters elections to those with applicable constituencies for selected constituencies
- After election continue, redirects directly to questions (constituency already selected)

ConstituencySelector testIds:
- Container: 'constituency-selector' (from data-testid on ConstituencySelector.svelte)
- Each constituency group renders a SingleGroupConstituencySelector with a select/radio input

Election accordion in results:
- Shows when `$dataRoot.elections.length > 1` (ALL elections, not just selected)
- data-testid="voter-results-election-select"

Constituency hierarchy:
- Constituencies have optional `parent` relation
- `election.getApplicableConstituency(selectedConstituencies)` finds the applicable constituency for an election given selected constituencies
- `ConstituencyGroup.getImpliedConstituency(selectedConstituencies)` resolves implied constituencies via parent chain

    The startfromcg overlay creates:
    - Same hierarchical structure as constituency overlay PLUS an orphan municipality with no parent region
    - The `startFromConstituencyGroup` setting reverses the flow: Constituencies -> Elections -> Questions

    **describe('startFromConstituencyGroup variant')**
    Serial describe block with shared page state.

    Use `test.use({ trace: 'off' })` for serial spec.

    Setup (beforeAll):
    - Create StrapiAdminClient, login
    - Query for the municipalities constituency group to get its database ID:
      ```
      const findResult = await client.findData('constituency-groups', { externalId: { $eq: 'test-cg-municipalities' } });
      const cgId = findResult.data[0].documentId;  // or .id depending on API response
      ```
    - Set `elections.startFromConstituencyGroup` to the constituency group's ID:
      ```
      await client.updateAppSettings({
        elections: {
          startFromConstituencyGroup: cgId,
          disallowSelection: false,
          showElectionTags: true
        },
        // ... all other sibling settings
      });
      ```

    1. `test('should show constituency selection first (reversed flow)')`:
       - Navigate Home -> start -> intro start
       - Should redirect to constituency selection page FIRST (not elections)

       - Verify the selector shows only municipalities (the specified constituency group)
       - Election selection should NOT be visible

    2. `test('should show election selection after constituency selection')`:
       - Select a municipality (e.g., muni-north-a)
       - Click continue (`testIds.voter.constituencies.continue`)
       - Should redirect to election selection page (elections filtered by applicable constituencies)
       - Verify `testIds.voter.elections.list` is visible
       - Select elections, click continue

    3. `test('should complete journey through questions to results')`:
       - After election selection, should proceed to questions
       - Answer all questions dynamically
       - Reach results page
       - Verify results are visible

    4. `test('should handle orphan municipality without error')`:
       - This can be a separate navigation test or combined with the main flow
       - Navigate Home -> start -> intro -> constituency selection
       - Select the orphan municipality (no parent region)
       - Click continue
       - Should show election selection (elections that have the orphan municipality's constituency group)
       - If no elections are applicable for the orphan, the election selector may show zero or filtered elections
       - The key assertion: no runtime error, no crash, the app gracefully handles the orphan

    afterAll:
    - Restore `elections.startFromConstituencyGroup` to undefined/null:

    IMPORTANT: The `startFromConstituencyGroup` setting value is a database ID (not externalId). Must query the constituency group first to get its actual ID. Use `client.findData('constituency-groups', { externalId: { $eq: 'test-cg-municipalities' } })` and extract the ID from the response.
