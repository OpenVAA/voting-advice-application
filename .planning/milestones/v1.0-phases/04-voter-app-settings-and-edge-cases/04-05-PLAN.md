  <action>
The nominations "when enabled" test navigates to `/en/nominations` and expects `voter-nominations-container` to be visible. This test fails because:

1. **Settings mutation safety**: The `beforeAll` calls `updateAppSettings` with only `entities` settings. Per Pitfall 2 (overwrite, not merge), this may overwrite other top-level settings. However, the nominations page only depends on `entities.showAllNominations` which is being set correctly.

2. **Possible root cause - concurrent test interference**: When running the full suite, other spec files (voter-settings, voter-popups) modify `entities` settings in their describe blocks. If those afterAll blocks don't properly restore `entities.showAllNominations: true`, the nominations test may run with `showAllNominations: false` and get redirected to home. The voter-settings.spec.ts `defaultQuestionSettings` restore does NOT include `entities` settings.

3. **Possible root cause - data loading**: The nominations `+layout.svelte` has an async data loading pattern. It shows `<Loading />` while waiting for `dataProvider.getNominationData()` to resolve. If the data request takes longer than the test timeout, or if it errors, the container won't appear.

4. **Possible root cause - entity-card testId mismatch**: The test locates entity cards via `nominationsList.getByTestId(testIds.voter.results.card)` which uses `entity-card`. The EntityCard component has `data-testid="entity-card"` hardcoded. However, the EntityList component that wraps them has `data-testid="entity-list"` (not `voter-nominations-list`). Check: the nominations page passes `data-testid="voter-nominations-list"` to EntityList via restProps, so EntityList renders with the overridden testId. But inside, the individual EntityCard components render with `data-testid="entity-card"`. This SHOULD work with scoped locator.

**Fix approach:**

A. Ensure the `beforeAll` includes ALL settings that the nomination page and other tests depend on, to prevent Pitfall 2 overwrites. Include `questions` settings alongside `entities` settings:

```typescript
test.beforeAll(async () => {
  await client.login();
  await client.updateAppSettings({
    entities: {
      showAllNominations: true,
      hideIfMissingAnswers: { candidate: false }
    },
    questions: {
      questionsIntro: { show: false, allowCategorySelection: false },
      categoryIntros: { show: false, allowSkip: true },
      showResultsLink: true
    }
  });
});
```

B. Add a longer timeout and explicit wait for the container to appear. The nomination data loading is async -- the layout shows `<Loading />` first, then swaps in the page content once data resolves. Increase the timeout to 15000ms and add a `waitForLoadState` before checking:

```typescript
test('should render nominations page with entries', async ({ page }) => {
  await page.goto(buildRoute({ route: 'Nominations', locale: 'en' }));
  await page.waitForLoadState('networkidle');

  // Wait for nominations container (layout loads data async, shows Loading first)
  await expect(page.getByTestId(testIds.voter.nominations.container)).toBeVisible({ timeout: 15000 });

  // Verify nominations list is visible
  await expect(page.getByTestId(testIds.voter.nominations.list)).toBeVisible();

  // Verify at least one entity card is present
  const nominationsList = page.getByTestId(testIds.voter.nominations.list);
  const entityCards = nominationsList.getByTestId(testIds.voter.results.card);
  await expect(entityCards.first()).toBeVisible({ timeout: 10000 });
  const cardCount = await entityCards.count();
  expect(cardCount).toBeGreaterThan(0);
});
```

C. Also fix the "when disabled" describe block to include complete sibling settings in both `beforeAll` and `afterAll`:

```typescript
// In "when disabled" beforeAll:
await client.updateAppSettings({
  entities: {
    showAllNominations: false,

  },
  questions: {
    questionsIntro: { show: false, allowCategorySelection: false },
    categoryIntros: { show: false, allowSkip: true },
    showResultsLink: true
  }
});

// In "when disabled" afterAll - restore defaults:
await client.updateAppSettings({
  entities: {
    showAllNominations: true,

  },
  questions: {
    questionsIntro: { show: false, allowCategorySelection: false },
    categoryIntros: { show: false, allowSkip: true },
    showResultsLink: true
  }
});
```

D. Add `test.describe.configure({ mode: 'serial' })` to the top-level nominations describe block to ensure "when enabled" runs before "when disabled" and their settings don't interfere.

Apply all fixes. Preserve the existing static pages describe block (about, info, privacy) unchanged since those tests pass.
