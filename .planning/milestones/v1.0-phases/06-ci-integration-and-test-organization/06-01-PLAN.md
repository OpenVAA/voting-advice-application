<task type="auto">
  <name>Task 1: Update e2e-tests job in GitHub Actions workflow</name>
  <files>.github/workflows/main.yaml</files>
  <action>
Make these five targeted changes to the `e2e-tests` job in `.github/workflows/main.yaml`:

1. **Add `--with-deps` to Playwright install** (line 160): Change `yarn playwright install` to `yarn playwright install --with-deps`. This installs OS-level browser dependencies (libgbm1, libnss3, etc.) on the Ubuntu runner that are required to launch Chromium headlessly.

2. **Override mock data generation in the "Configure environment" step** (line 130): After `cp .env.example .env`, add a second line: `sed -i 's/GENERATE_MOCK_DATA_ON_INITIALISE=true/GENERATE_MOCK_DATA_ON_INITIALISE=false/' .env`. The new test structure imports its own deterministic datasets via the Admin Tools API; mock data generation would pollute the database and cause test failures.

3. **Remove the 30-second sleep step** (lines 183-186): Delete the entire step:
   ```yaml
   - name: "Sleep for 30s to allow mock data generation to complete before running e2e tests"
     uses: juliangruber/sleep-action@v2
     with:
       time: 30s
   ```
   This step exists solely to wait for mock data generation, which is now disabled. The `yarn dev:start` command already uses `docker compose up --wait` which waits for healthcheck conditions.

4. **Remove the stale playwright-setup-report upload step** (lines 191-197): Delete the entire step:
   ```yaml
   - name: "Upload playwright setup report"
     uses: actions/upload-artifact@v4
     if: always()
     with:
       name: playwright-setup-report
       path: tests/playwright-setup-report/
       retention-days: 30
   ```
   This was for the old globalSetup pattern. With the project dependencies pattern, setup project results appear in the main HTML report at `tests/playwright-report/`.

5. **Keep the existing HTML report upload step unchanged** (lines 199-205): The `playwright-report` artifact upload already uses the correct path (`tests/playwright-report/`) and configuration (`if: always()`, `retention-days: 30`). Verify it is intact after the deletions above. This satisfies CI-02.

Do NOT change the `frontend-and-shared-module-validation` or `backend-validation` jobs. Do NOT change the trigger configuration (`on:` block). Do NOT add matrix strategies or additional jobs.
