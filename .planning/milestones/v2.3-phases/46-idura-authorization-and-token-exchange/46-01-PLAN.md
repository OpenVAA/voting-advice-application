<objective>
Implement the Idura-specific OIDC authorization and token exchange logic (JAR + private_key_jwt), create the server-side authorize endpoint, and update the token route to use the provider abstraction.

Purpose: This is the core security implementation for Idura FTN compliance. JAR (RFC 9101) is required for all FTN authorization requests as of Aug 2025, and private_key_jwt replaces client_secret for token exchange. The server-side authorize endpoint ensures signing keys never reach the browser.

Output: Working Idura provider methods (getAuthorizeUrl, exchangeCodeForToken), new /api/oidc/authorize endpoint, updated /api/oidc/token endpoint using provider abstraction
</objective>

**`getAuthorizeUrl({ redirectUri })` implementation (per D-01, D-02, D-03, D-04):**

1. Read from server constants: `IDURA_SIGNING_JWKS`, `IDURA_SIGNING_KEY_KID`, `IDURA_DOMAIN`
2. Read from public constants: `PUBLIC_IDENTITY_PROVIDER_CLIENT_ID`
3. Parse the signing JWKS: `const signingJwkSet = JSON.parse(constants.IDURA_SIGNING_JWKS || '[]') as jose.JWK[]`
4. Find the signing key by KID: `const signingJwk = signingJwkSet.find(jwk => jwk.kid === constants.IDURA_SIGNING_KEY_KID)`
5. If no signing key found, throw `new Error('Idura signing key not found for kid: ' + constants.IDURA_SIGNING_KEY_KID)`
6. Import the key: `const signingKey = await jose.importJWK(signingJwk, 'RS256')`
7. Generate state and nonce: `const state = crypto.randomUUID()` and `const nonce = crypto.randomUUID()`
8. Build the signed JAR (RFC 9101) using `jose.SignJWT`:
   ```
   const requestObject = await new jose.SignJWT({
     response_type: 'code',
     response_mode: 'query',
     client_id: clientId,
     redirect_uri: redirectUri,
     scope: 'openid profile',
     state,
     nonce,
     iss: clientId,
     aud: `https://${constants.IDURA_DOMAIN}`
   })
     .setProtectedHeader({ alg: 'RS256', kid: signingJwk.kid! })
     .sign(signingKey);
   ```
9. Build the authorize URL: `https://${constants.IDURA_DOMAIN}/oauth2/authorize?client_id=${encodeURIComponent(clientId)}&request=${requestObject}`
10. Return: `{ authorizeUrl, clientSideRedirect: false, state, nonce }`

IMPORTANT: The return type `AuthorizeResult` from Phase 45 only has `authorizeUrl` and `clientSideRedirect`. We need to also return `state` and `nonce` so the authorize endpoint can store them in cookies. To handle this without modifying Phase 45 types, extend the return with additional properties: `return { authorizeUrl, clientSideRedirect: false, state, nonce } as AuthorizeResult & { state: string; nonce: string }`. Alternatively, if the Phase 45 `AuthorizeResult` type already includes optional `state` and `nonce` fields, use those. If not, cast the return and the authorize route will access the extra properties.

Actually, the cleaner approach: Update the `AuthorizeResult` type in `types.ts` to include optional `state?: string` and `nonce?: string` fields (they are optional because Signicat uses client-side PKCE and doesn't return them from this method). This is a minor type change that Phase 45 should have anticipated.

**`exchangeCodeForToken({ authorizationCode, redirectUri })` implementation (per D-08, D-09):**

1. Read from server constants: `IDURA_SIGNING_JWKS`, `IDURA_SIGNING_KEY_KID`, `IDURA_DOMAIN`
2. Read from public constants: `PUBLIC_IDENTITY_PROVIDER_CLIENT_ID`
3. Parse signing JWKS and find signing key (same as getAuthorizeUrl)
4. Import the key: `const signingKey = await jose.importJWK(signingJwk, 'RS256')`
5. Build the token endpoint URL: `const tokenEndpoint = 'https://' + constants.IDURA_DOMAIN + '/oauth2/token'`
6. Build the client assertion JWT:
   ```
   const clientAssertion = await new jose.SignJWT({})
     .setProtectedHeader({ alg: 'RS256', kid: signingJwk.kid! })
     .setIssuer(clientId)
     .setSubject(clientId)
     .setAudience(tokenEndpoint)
     .setExpirationTime('5m')
     .setJti(crypto.randomUUID())
     .sign(signingKey);
   ```
7. POST to token endpoint:
   ```
   const response = await fetch(tokenEndpoint, {
     method: 'POST',
     headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
     body: new URLSearchParams({
       grant_type: 'authorization_code',
       code: authorizationCode,
       redirect_uri: redirectUri,
       client_assertion_type: 'urn:ietf:params:oauth:client-assertion-type:jwt-bearer',
       client_assertion: clientAssertion
     }).toString()
   });
   ```
8. If `!response.ok`, throw `new Error('Idura token exchange failed: ' + response.status)`
9. Parse response: `const { id_token } = await response.json()`
10. Return: `{ idToken: id_token }`

Implement `POST` handler:

1. Parse request body: `const { redirectUri, codeChallenge } = await request.json()`
2. Validate: if `!redirectUri`, return `error(400, { message: 'redirectUri is required' })`
3. Call provider: `const provider = getActiveProvider()`
4. Get authorize result: `const result = await provider.getAuthorizeUrl({ redirectUri, codeChallenge })`
5. If the provider returns `state` and `nonce` (Idura does, Signicat does not), store them in httpOnly cookies:
   ```typescript
   if (result.state) {
     cookies.set('oidc_state', result.state, {
       httpOnly: true,
       secure: true,
       sameSite: 'lax',  // 'lax' required for cross-origin IdP redirects
       path: '/',
       maxAge: 600  // 10 minutes
     });
   }
   if (result.nonce) {
     cookies.set('oidc_nonce', result.nonce, {
       httpOnly: true,
       secure: true,
       sameSite: 'lax',
       path: '/',
       maxAge: 600
     });
   }
   ```
6. Return: `json({ authorizeUrl: result.authorizeUrl })`

Wrap the provider call in try/catch. On error, log the error and return `error(500, { message: 'Failed to construct authorization request' })`.
