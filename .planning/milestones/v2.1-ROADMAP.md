## Phase Details

### Phase 40: Email and Auth Flow Fixes
**Goal**: Candidate registration and password reset email flows complete end-to-end in E2E tests
**Depends on**: Phase 39
**Requirements**: EMAIL-01, EMAIL-02, AUTH-01
**Success Criteria** (what must be TRUE):
  1. The `candidate-registration` E2E test for email link flow passes: invite is sent, Mailpit captures it, link is clicked, and the candidate session is established
  2. The `candidate-registration` E2E test for password reset flow passes: reset is requested, Mailpit captures the email, link is clicked, and a new password is set successfully
  3. The `candidate-profile` E2E test for fresh candidate registration passes (depends on working email/auth flow from criteria 1)
  4. No test infrastructure workarounds that mask real failures (e.g., skipped assertions, hardcoded tokens)
**Plans**: TBD

### Phase 41: Popup Timing Fix
**Goal**: Feedback popup appears reliably on the results page after its configured delay
**Depends on**: Nothing (independent of Phase 40)
**Requirements**: POP-01
**Success Criteria** (what must be TRUE):
  1. The `voter-popups` E2E test for feedback popup timing passes: popup appears after the configured delay on the results page
  2. Popup does not appear prematurely or fail to appear due to race conditions
**Plans**: TBD
