  <action>
    The root cause: after a form-action login redirect (303), the Vite dev server's SvelteKit client-side router retains stale streaming state for the protected layout's universal load function. `page.reload()` does a browser-level reload but the SvelteKit hydration still gets stuck waiting for promises that were part of the original form-action response.

    Fix: replace `page.reload({ waitUntil: 'domcontentloaded' })` with `page.goto(buildRoute({ route: 'CandAppHome', locale: 'en' }), { waitUntil: 'networkidle' })`. This is a fresh top-level navigation from Playwright's perspective -- it completely tears down the current page and creates a new navigation context, bypassing any stale form-action streaming state. Using `waitUntil: 'networkidle'` ensures the protected layout's API calls complete before proceeding.
