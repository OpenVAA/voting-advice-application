### Pattern: D-07 bank-auth inline-justified skip (RESEARCH §"Example 1" + CONTEXT D-07)

```typescript
// Source: candidate-bank-auth.spec.ts:196-199 (current state — this task amends)
test('should return session with magic link when candidate is created', async () => {
  // This test only runs meaningfully when Edge Function keys are configured
  // Skip if no user was created in the previous test
  // reason: bank-auth is opt-in via @bank-auth tag (env-gated; PLAYWRIGHT_BANK_AUTH=1
  //   selects the project per playwright.config.ts; disabled by default in CI). This skip
  //   is a precondition-gate for the Edge Function integration test, NOT a race —
  //   converting to expect.poll would mask "Edge Function keys not configured" → false-positive timeout.
  // eslint-disable-next-line playwright/no-skipped-test
  test.skip(!createdUserId, 'Skipped: Edge Function keys not configured for full integration');
  // ...
});
```

Key rules per D-07:
1. The `// reason:` block goes IMMEDIATELY ABOVE the `// eslint-disable-next-line` directive (not on a single line)
2. The `// reason:` text must:
   - State that bank-auth is opt-in (via the @bank-auth tag in playwright.config.ts)
   - State that the skip is a precondition-gate, not a race
   - State why expect.poll is wrong here (would mask config-missing as timeout)
3. The directive scope is `playwright/no-skipped-test` ONLY (not a blanket disable)
4. The skip predicate stays as `!createdUserId` (or the equivalent precondition variable) — do NOT convert to `test.skip(true, ...)`

### Pattern: candidate-bank-auth.spec.ts non-skip warnings (16 warnings)

Per Plan 1 inventory: 12 no-conditional-expect + 4 no-conditional-in-test = 16 sites in this file (excluding the 1 skip).

The bank-auth file's nature (JWE/JWT crypto + Edge Function calls) means:
- Many `if (await ...).catch(...)` patterns trying to handle "Edge Function may not be reachable" → these are precondition-gates dressed as conditionals. Convert via the same pattern as D-07: gate the entire test or describe block with `test.skip(condition, ...)` (with full inline justification per the D-07 shape) AND remove the conditional from inside the test body.

  ```typescript
  // BEFORE (synthesis from typical bank-auth shape)
  test('should fetch user data', async () => {
    if (!edgeFunctionAvailable) return;
    const data = await fetchUserData();
    if (data) {
      expect(data.id).toBe(createdUserId);
    }
  });

  // AFTER
  test('should fetch user data', async () => {
    // reason: gates the same Edge Function precondition as the line-199 skip;
    //   this is precondition-not-met, not a race
    // eslint-disable-next-line playwright/no-skipped-test
    test.skip(!edgeFunctionAvailable, 'Edge Function not available');
    const data = await fetchUserData();
    expect(data).not.toBeNull();        // unconditional
    expect(data!.id).toBe(createdUserId); // unconditional
  });
  ```

  This shape produces 1 additional inline-justified `// reason:` per gating site. Each gating decision is per-test — review whether the precondition is genuine (Edge Function availability) or a race-mask (data not yet loaded). The former gets D-07 treatment; the latter gets unconditional expect.poll / waitFor.

- Some `try { ... } catch (error) { expect(error)... }` patterns capturing real errors → use the `getError` wrapper from Pattern 5 verbatim.

- Some `if (await x.isVisible())` masks → unconditional `await expect(x).toBeVisible(); expect(...);`.

The exact distribution across these three sub-patterns is per-site judgment per the Plan 1 inventory's failure-type classification.
