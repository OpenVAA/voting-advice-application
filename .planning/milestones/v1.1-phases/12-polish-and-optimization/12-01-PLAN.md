2. Define catalog in .yarnrc.yml (add after existing config):
   ```yaml

     typescript: ^5.7.3
     vitest: ^2.1.8
     eslint: ^9.39.2
     prettier: ^3.7.4
     tsx: ^4.19.2
     "@typescript-eslint/eslint-plugin": ^8.19.1
     "@typescript-eslint/parser": ^8.19.1
     eslint-plugin-simple-import-sort: ^12.1.1
     eslint-plugin-import: ^2.32.0
     "@eslint/eslintrc": ^3.2.0
     "@eslint/js": ^9.39.1
     globals: ^15.14.0
   ```

   Notes on version choices:
   - `prettier`: Use ^3.7.4 (normalize to highest version across workspaces, from docs)
   - `@eslint/js`: Use ^9.39.1 (normalize to highest, from docs)
   - Do NOT catalog `vitest` in apps/docs (it uses ^4.0.15, different major) -- leave docs explicit
   - Do NOT catalog `globals` in apps/docs (it uses ^16.5.0, different major) -- leave docs explicit
   - Do NOT include `@openvaa/shared-config: "workspace:^"` in catalog -- test this first. If `workspace:^` works in catalog, include it; if not, keep explicit. Per research pitfall 2, this may cause issues.

- Replace `"eslint": "^9.39.2"` with `"eslint": "catalog:"` in shared-config, frontend, strapi, root
- Replace `"prettier": "^3.4.2"` (or ^3.7.4) with `"prettier": "catalog:"` in all workspaces EXCEPT apps/docs if docs has ^3.7.4 already (normalize; docs can also use catalog since we set catalog to ^3.7.4)
- Replace `"tsx": "^4.19.2"` (or ^4.19.3) with `"tsx": "catalog:"` in root, frontend, data
- Replace ESLint-related deps (`@typescript-eslint/*`, `eslint-plugin-*`, `@eslint/*`) with `"catalog:"` where they appear

- Any dependency not in the catalog list
