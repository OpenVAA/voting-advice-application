  <action>
Create `.github/workflows/release.yml` with the following content:

```yaml
name: Release

on:
  push:
    branches:
      - main

concurrency: ${{ github.workflow }}-${{ github.ref }}

jobs:
  release:
    name: Version Packages
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write

    steps:
      - name: Checkout Repo
        uses: actions/checkout@v4

      - name: Setup Yarn 4.6
        uses: threeal/setup-yarn-action@v2
        with:
          version: 4.6

      - name: Setup Node.js 20.18.1
        uses: actions/setup-node@v4
        with:
          node-version: 20.18.1
          cache: "yarn"

      - name: Install Dependencies
        run: yarn install --frozen-lockfile

      - name: Create Release Pull Request
        uses: changesets/action@v1
        with:
          title: "chore: version packages"
          commit: "chore: version packages"
          # Uncomment when ready to publish (Phase 11)
          # publish: yarn changeset publish
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Key design decisions:
- `concurrency` key serializes runs to prevent race conditions when multiple PRs merge quickly (Pitfall 6 from research)
- Explicit `permissions` for `contents: write` and `pull-requests: write` to avoid GITHUB_TOKEN permission issues (Pitfall 3 from research)
- `GITHUB_TOKEN` env var is passed to changesets/action so the `@changesets/changelog-github` plugin can look up PR numbers for changelog entries (Pitfall 4 from research)
- Node/Yarn setup matches the existing CI workflow pattern in `main.yaml` (same versions: Node 20.18.1, Yarn 4.6)
- The `publish` line is commented out with a clear "Phase 11" note per user decision
- No `yarn build` step needed — `changeset version` only manipulates package.json and CHANGELOG.md files, no compilation required
- The Version Packages PR requires manual review before merge per user decision (no auto-merge configured)
