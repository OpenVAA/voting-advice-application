<task type="auto">
  <name>Task 2: Fresh install verification for all 4 publishable packages</name>
  <files></files>
  <action>
1. Pack all 4 publishable packages into tarballs:
   ```bash
   cd packages/core && yarn pack -o /tmp/openvaa-core.tgz
   cd packages/data && yarn pack -o /tmp/openvaa-data.tgz
   cd packages/matching && yarn pack -o /tmp/openvaa-matching.tgz
   cd packages/filters && yarn pack -o /tmp/openvaa-filters.tgz
   ```

2. Inspect each tarball to verify:
   - Contains `dist/index.js` and `dist/index.d.ts`
   - Contains `LICENSE`
   - Does NOT contain `src/` directory
   - Does NOT contain `node_modules/`
   - package.json inside tarball has no `workspace:^` strings

   Use `tar tzf /tmp/openvaa-core.tgz` to list contents and `tar xf /tmp/openvaa-core.tgz -O package/package.json | grep workspace` to check for workspace protocol.

3. Create a temporary test project:
   ```bash
   TESTDIR=$(mktemp -d)
   cd "$TESTDIR"
   npm init -y
   ```

4. Install tarballs. Note: filters depends on data depends on core, and matching depends on core. Install in dependency order:
   ```bash
   npm install /tmp/openvaa-core.tgz
   npm install /tmp/openvaa-data.tgz
   npm install /tmp/openvaa-matching.tgz
   npm install /tmp/openvaa-filters.tgz
   ```

5. Add `"type": "module"` to the test project's package.json (since packages are ESM-only).

6. Test imports work:
   ```bash
   node -e "import('@openvaa/core').then(m => { console.log('core OK:', Object.keys(m).length, 'exports'); }).catch(e => { console.error('core FAIL:', e.message); process.exit(1); })"
   node -e "import('@openvaa/data').then(m => { console.log('data OK:', Object.keys(m).length, 'exports'); }).catch(e => { console.error('data FAIL:', e.message); process.exit(1); })"
   node -e "import('@openvaa/matching').then(m => { console.log('matching OK:', Object.keys(m).length, 'exports'); }).catch(e => { console.error('matching FAIL:', e.message); process.exit(1); })"
   node -e "import('@openvaa/filters').then(m => { console.log('filters OK:', Object.keys(m).length, 'exports'); }).catch(e => { console.error('filters FAIL:', e.message); process.exit(1); })"
   ```

7. Clean up:
   ```bash
   rm -rf "$TESTDIR" /tmp/openvaa-*.tgz
   ```

If any import fails, investigate the error. Common issues:
- Missing dependency in package.json (workspace dep not listed in dependencies)
- Incorrect exports field path
- ESM import resolution issues (missing file extensions in output)
  </action>
  <verify>
