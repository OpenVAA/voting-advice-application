<task type="auto">
  <name>Task 2: Configure Changesets public access and release workflow publishing</name>
  <files>
    .changeset/config.json,

    package.json
  </files>
  <action>
1. Update `.changeset/config.json`:
   - Change `"access": "restricted"` to `"access": "public"`

2. Update `.github/workflows/release.yml`:
   - Replace the commented-out publish line with the custom publish command:
     ```yaml
     - name: Create Release Pull Request or Publish
       uses: changesets/action@v1
       with:
         title: "chore: version packages"
         commit: "chore: version packages"
         publish: yarn release
       env:
         GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
         NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
     ```
   - Add `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}` to env as well (some npm configurations look for this)
   - Add `registry-url: 'https://registry.npmjs.org'` to the `setup-node` step

3. Add `release` script to root `package.json`:
   ```json
   "release": "yarn build && yarn workspaces foreach --no-private yarn npm publish --access public --tolerate-republish"
   ```
   Verify exact `yarn workspaces foreach` syntax by running `yarn workspaces foreach --help` first. The `--no-private` flag filters out packages with `"private": true`. The `--tolerate-republish` flag prevents errors if a version is already published.

4. Verify `yarn pack --dry-run` on each publishable package shows no `workspace:^` strings in the tarball package.json. Run:
   ```bash
   cd packages/core && yarn pack --dry-run
   cd packages/data && yarn pack --dry-run
   cd packages/matching && yarn pack --dry-run
   cd packages/filters && yarn pack --dry-run
   ```
   Inspect the output -- `workspace:^` strings should be resolved to real version numbers by Yarn's packing process.
  </action>

  </verify>
