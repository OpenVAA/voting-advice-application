## Phase Details

### Phase 27: Candidate Route Migration
**Goal**: All candidate app routes are fully Svelte 5 idiomatic — no legacy reactive statements, event directives, or lifecycle imports remain
**Depends on**: Phase 26 (v1.3 complete)
**Requirements**: ROUTE-01, ROUTE-02, ROUTE-03, ROUTE-04, EVNT-01, EVNT-02, LIFE-01
**Success Criteria** (what must be TRUE):
  1. Every auth route file (login, register, password) uses $derived/$effect instead of $: statements
  2. Every protected route file (profile, questions, settings, preview) uses $derived/$effect instead of $: statements
  3. Every preregister route file uses $derived/$effect instead of $: statements
  4. Every candidate layout file uses $derived/$effect instead of $: statements
  5. All on:event directives and |modifier syntax are replaced with native event attributes and explicit handler calls in candidate routes
**Plans:** 4/4 plans complete
Plans:
- [x] 27-01-PLAN.md — Migrate 10 minimal-complexity files (layouts to snippets, event directive, simple pages)
- [x] 27-02-PLAN.md — Migrate 7 auth/preregister pages ($derived, $effect, $page to page)
- [x] 27-03-PLAN.md — Migrate 4 moderate-complexity files (root layout, settings, preview, home)
- [x] 27-04-PLAN.md — Migrate 4 high-complexity files (profile, questions layout, [questionId], protected layout)

### Phase 28: Validation Gate
**Goal**: The candidate app migration is verified — no legacy patterns remain, TypeScript is clean, and the full E2E suite passes
**Depends on**: Phase 27
**Requirements**: VALD-01, VALD-02, VALD-03
**Success Criteria** (what must be TRUE):
  1. A codebase search for $:, on:event directives, <slot>, and createEventDispatcher finds zero matches in candidate app routes
  2. TypeScript compilation reports zero errors in the candidate app
  3. All 5 candidate-app E2E test files execute and pass
**Plans:** 3/3 plans complete
Plans:
- [x] 28-01-PLAN.md — Legacy pattern audit + TypeScript check (VALD-01, VALD-02)
- [x] 28-02-PLAN.md — SES email infrastructure fix + candidate E2E suite (VALD-03 partial)
- [x] 28-03-PLAN.md — Gap closure: fix 2 failing E2E tests (Vite dev-mode streaming workaround) (VALD-03)
