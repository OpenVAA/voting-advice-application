# Requirements: OpenVAA Branch Integration

**Defined:** 2026-03-22
**Core Value:** A reliable, well-tested VAA framework that developers can confidently extend, customize, and deploy for real elections.

## v2.0 Requirements

Requirements for integrating feat-gsd-supabase-migration into the Svelte 5 codebase. Each maps to roadmap phases.

### Backend

- [x] **BACK-01**: apps/supabase/ workspace integrated with schema, migrations, seed data, Edge Functions, and pgTAP tests
- [x] **BACK-02**: @openvaa/supabase-types package integrated with Database types, COLUMN_MAP, PROPERTY_MAP, TABLE_MAP
- [x] **BACK-03**: Schema reorganization completed (numbered files, p_ parameter prefixes, public. qualifiers)
- [x] **BACK-04**: @supabase/supabase-js added to workspace dependencies via Yarn catalog

### Adapter

- [x] **ADPT-01**: SupabaseDataProvider integrated with all 7 read methods, adapted to Svelte 5 and apps/frontend/ paths

- [x] **ADPT-03**: SupabaseAdminWriter integrated (question custom data, job management) adapted to Svelte 5
- [x] **ADPT-04**: SupabaseFeedbackWriter integrated adapted to Svelte 5
- [x] **ADPT-05**: Supabase adapter mixin and utility functions (mapRow, getLocalized, localizeRow, toDataObject, storageUrl) integrated
- [x] **ADPT-06**: Dynamic adapter switch wired to select Supabase adapter

### Auth

- [x] **AUTH-03**: Auth context rewritten for Supabase (isAuthenticated replacing authToken)
- [x] **AUTH-04**: Browser and server Supabase client setup (lib/supabase/)
- [x] **AUTH-05**: Auth API routes (callback, logout) adapted to Svelte 5
- [x] **AUTH-06**: Candidate preregister route with Edge Function invocation, adapted to Svelte 5
- [x] **AUTH-07**: Protected layout guards using Supabase sessions

### Tests

- [x] **TEST-01**: SupabaseAdminClient replacing StrapiAdminClient with all 14 methods
- [x] **TEST-02**: Test datasets converted to snake_case format with project_id/published fields
- [x] **TEST-03**: All spec files updated for Supabase method signatures and data format
- [x] **TEST-04**: Email helper rewritten for Inbucket REST API (replacing SES)
- [x] **TEST-05**: Playwright config updated for Supabase backend (workers, baseURL, projects)
- [x] **TEST-06**: 14 remaining E2E failures from schema reorg fixed
- [x] **TEST-07**: All FIXMEs and TODOs in E2E test files collected and addressed

### CI/Docs

## Out of Scope

| Feature | Reason |
|---------|--------|
| New feature development | Integration only — no new capabilities beyond what exists on either branch |
| Svelte 5 migration of parallel branch code beyond adapter/auth/routes | Only files that touch frontend integration need Svelte 5 adaptation |
| Admin app UI | Separate milestone (ADMIN-01/02/03) |
| GraphQL via pg_graphql | No current frontend need |
| Supabase Realtime | No current use case |
| Schema-per-tenant isolation | Disproportionate overhead for 10-50 tenants |
| Architect/Components/LLM skills | Deferred to post-integration stabilization |
