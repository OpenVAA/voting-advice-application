# Requirements: OpenVAA Framework Evolution

**Defined:** 2026-03-20
**Core Value:** A reliable, well-tested VAA framework that developers can confidently extend, customize, and deploy for real elections.

## v1.4 Requirements

Requirements for Svelte 5 Migration (Candidate App). Each maps to roadmap phases.

### Route Migration

- [x] **ROUTE-01**: All candidate auth routes (login, register, password) use $derived/$effect instead of $: reactive statements
- [x] **ROUTE-02**: All candidate protected routes (profile, questions, settings, preview) use $derived/$effect instead of $: reactive statements
- [x] **ROUTE-03**: All candidate preregister routes use $derived/$effect instead of $: reactive statements
- [x] **ROUTE-04**: All candidate layouts use $derived/$effect instead of $: reactive statements

### Event Migration

- [x] **EVNT-01**: All on:event directives in candidate routes replaced with native event attributes (onclick, onsubmit, etc.)
- [x] **EVNT-02**: All event modifiers (|preventDefault, |stopPropagation) replaced with explicit handler calls

### Lifecycle Migration

- [x] **LIFE-01**: All onMount/onDestroy in candidate routes converted to $effect where appropriate

### Validation

- [x] **VALD-01**: Zero legacy Svelte 4 patterns ($:, on:event, <slot>, createEventDispatcher) in candidate routes
- [x] **VALD-02**: Zero TypeScript errors in candidate app
- [x] **VALD-03**: All 5 candidate-app E2E test files pass

## Future Requirements

### Context System Rewrite

- **CTX-01**: Data context rewritten with Svelte 5 native reactivity ($state/$derived)
- **CTX-02**: Page datum stores replaced with Svelte 5 subscription patterns
- **CTX-03**: Admin job stores updated with Svelte 5 patterns
