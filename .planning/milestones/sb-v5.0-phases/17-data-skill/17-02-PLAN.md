**# Extension Patterns for @openvaa/data** (title)

**## Adding a New Entity Type** (~60 lines)
Step-by-step guide listing every file that must be created or modified. Use Candidate as the reference implementation. Number each step. For each step specify: the file path, what to do, and what to look at for the pattern.

Steps (all paths relative to `packages/data/src/`):

1. **Create type file** `objects/entities/variants/{name}.type.ts`
   - Define `{Name}Data extends EntityData` interface
   - Pattern: copy `objects/entities/variants/candidate.type.ts`, adapt fields
   - Import from `../../../internal`

2. **Create class file** `objects/entities/variants/{name}.ts`
   - Class `{Name} extends Entity<typeof ENTITY_TYPE.{Name}, {Name}Data>`
   - Add `readonly objectType = OBJECT_TYPE.{Name}`
   - Implement `DataAccessor<{Name}Data>` with typed property getters using smart defaults
   - Pattern: copy `objects/entities/variants/candidate.ts`
   - Import from `../../../internal`

3. **Add to ENTITY_TYPE** `objects/entities/base/entityTypes.ts`
   - Add `{Name}: '{name}'` to the `ENTITY_TYPE` const
   - The `EntityType` union type auto-derives from the const

4. **Add to OBJECT_TYPE** `core/objectTypes.ts`
   - Add `{Name}: '{name}'` to `OBJECT_TYPE` const
   - Add `[OBJECT_TYPE.{Name}]: {Name}` to `ObjectTypeMap`
   - Note the comment: "NB! When editing these, be sure to update `/utils/typeGuard.ts` as well."

5. **Update type guards** `utils/typeGuards.ts`
   - Add `obj.objectType === OBJECT_TYPE.{Name}` to `isEntity()` function's condition

6. **Update EntityVariant** `objects/entities/variants/variants.ts`
   - Add to `EntityVariant` type map: `[ENTITY_TYPE.{Name}]: {Name}`
   - Add to `EntityVariantData` type map: `[ENTITY_TYPE.{Name}]: {Name}Data`
   - Update `parseEntityTree` if tree format needs new variant handling

7. **Add exports to internal.ts** `internal.ts`
   - Add `export * from './objects/entities/variants/{name}.type'` AFTER the last entity type export
   - Add `export * from './objects/entities/variants/{name}'` AFTER the type export
   - Position: in the entity variants block (after alliance, before nominations)

8. **Add to index.ts** `index.ts`
   - Add `{Name}Data` to the type exports block
   - Add `{Name}` to the class exports block

9. **Add DataRoot collection** `root/dataRoot.ts` (only if entity needs its own collection getter)
   - Add `get {name}s()` collection getter following the pattern of `get candidates()`
   - Add `get{Name}(id)` id getter following the pattern of `getCandidate(id)`
   - Add private `_{name}s` MappedCollection field

10. **Create co-located test** `objects/entities/variants/{name}.test.ts`
    - Import from `../../../internal` and `../../../testUtils`
    - Test that all provided data is accessible via getters
    - Pattern: copy `objects/entities/variants/candidate.test.ts`

**## Adding a New Question Type** (~70 lines)
Step-by-step guide. Use SingleChoiceOrdinalQuestion as the matchable reference and TextQuestion as the non-matchable reference.

Steps (all paths relative to `packages/data/src/`):

1. **Create class file** `objects/questions/variants/{name}Question.ts`
   - For simple (non-choice) questions: extend `Question<typeof QUESTION_TYPE.{Name}, {Value}>`
   - For single-choice questions: extend `SingleChoiceQuestion<typeof QUESTION_TYPE.{Name}, {ChoiceValue}>`
   - For multiple-choice questions: extend `MultipleChoiceQuestion<typeof QUESTION_TYPE.{Name}, {ChoiceValue}>`
   - Add `readonly objectType = OBJECT_TYPE.{Name}Question`
   - If matchable: set `get isMatchable() { return true; }` and implement `_normalizeValue()`
   - If non-matchable: `isMatchable` defaults to `false`, no `_normalizeValue()` needed
   - Pattern for matchable: `objects/questions/variants/singleChoiceOrdinalQuestion.ts`
   - Pattern for non-matchable: `objects/questions/variants/textQuestion.ts`
   - Import from `../../../internal`

2. **Create type file** (only if question has extra data fields beyond QuestionData)
   - `objects/questions/variants/{name}Question.type.ts`
   - Define `{Name}QuestionData extends QuestionData` (or `extends ChoiceQuestionData<{ChoiceValue}>` for choice questions)
   - Pattern: `objects/questions/variants/numberQuestion.type.ts` (has min/max fields)

3. **Add to QUESTION_TYPE** `objects/questions/base/questionTypes.ts`
   - If simple: add to `SIMPLE_QUESTION_TYPE`
   - If single-choice: add to `SINGLE_CHOICE_QUESTION_TYPE`
   - If multiple-choice: add to `MULTIPLE_CHOICE_QUESTION_TYPE`
   - The composite consts (`CHOICE_QUESTION_TYPE`, `QUESTION_TYPE`) auto-derive from spreads

4. **Add to OBJECT_TYPE** `core/objectTypes.ts`
   - Add `{Name}Question: '{name}Question'` to `OBJECT_TYPE` const
   - Add `[OBJECT_TYPE.{Name}Question]: {Name}Question` to `ObjectTypeMap`

5. **Update type guards** `utils/typeGuards.ts`
   - If it's a ChoiceQuestion subtype, add to `isChoiceQuestion()` condition
   - If it's a SingleChoiceQuestion, add to `isSingleChoiceQuestion()` condition
   - If it's a MultipleChoiceQuestion, add to `isMultipleChoiceQuestion()` condition
   - `isQuestion()` uses `.endsWith('Question')` so no change needed there

6. **Update QuestionVariant** `objects/questions/variants/variants.ts`
   - Add to `QUESTION_VARIANT` map: `[QUESTION_TYPE.{Name}]: {Name}Question` (or use class reference)
   - `createQuestion()` uses `QUESTION_VARIANT` map to select the class -- no additional changes needed

7. **Update AnswerValue** `objects/questions/base/answer.type.ts`
   - Add `[QUESTION_TYPE.{Name}]: {ValueType}` to the `AnswerValue` mapped type

8. **Add exports to internal.ts** `internal.ts`
   - Add type file export (if created) in the question variants block
   - Add class file export after the type
   - Position: in the question variants block (after existing question variants, before the `variants.ts` re-export)

9. **Add to index.ts** `index.ts`
   - Add data type (if any) to type exports
   - Add class to class exports

10. **Create co-located test** `objects/questions/variants/{name}Question.test.ts`
    - Test value validation via `ensureValue()`
    - Test answer formatting via `getFormattedAnswer()` if applicable
    - If matchable: test `normalizeValue()` returns correct coordinates
    - Import from `../../../internal` and `../../../testUtils`

**## Adding a New Nomination Variant** (~25 lines)
Abbreviated guide since nomination variants follow the same pattern as entity variants but with Nomination base class. Reference: `candidateNomination.ts` for simple nomination, `organizationNomination.ts` for nomination with child nominations.

Key differences from entity extension:
- Class extends `Nomination<{EntityType}, {ParentNominationType}, {Name}NominationData>`
- The three generic params are: entity variant it links to, parent nomination variant (or `never`), and its own data type
- Must update `NominationVariant` type map in `objects/nominations/variants/variants.ts`
- May need auto-generated (implied) entity handling if this is a grouping nomination

**## Verification After Extension** (~15 lines)
After completing any extension, verify:
1. `cd packages/data && yarn test:unit` -- all existing tests pass
2. New test file passes
3. `yarn build:app-shared` -- builds without circular dependency errors
4. Check `internal.ts` export ordering: base types before dependent types
5. Check that all OBJECT_TYPE values are unique strings (no collisions)
6. Check that `typeGuards.ts` includes the new type where relevant
7. If matchable question: verify `normalizeValue()` returns correct dimensionality (check `normalizedDimensions` getter)
