    ```
    # Extension Patterns for @openvaa/filters

    Step-by-step guides for the most common filter package extensions. All file paths in steps are relative to `packages/filters/src/` unless stated otherwise.
    ```

    **Guide 1: Adding a New Filter Type** (base class + concrete filter)

    Reference implementation: `NumberQuestionFilter` (see `filter/number/numberQuestionFilter.ts`) extending `NumberFilter` base.

    Steps (numbered, with exact file paths relative to `packages/filters/src/`):

    1. **Choose or create a base class** `filter/{category}/`
       - Extend one of the 3 existing bases: `EnumeratedFilter` (include/exclude sets), `NumberFilter` (min/max/excludeMissing), or `TextFilter` (text include/exclude/caseSensitive)
       - To create a new base: create `filter/{category}/{category}Filter.ts` extending `Filter<TTarget, TValue>` from `filter/base/filter.ts`
       - Implement abstract methods: `testValue(value)` (single value test), `testValues(values)` (multi value test), override `_rules` getter/setter
       - Pattern: copy `filter/number/numberFilter.ts` for a new base

    2. **Create concrete filter class** `filter/{category}/{name}Filter.ts`
       - For question-based: extend chosen base, accept `FilterableQuestion` subtype in constructor via `QuestionFilterOptions`
       - For property-based: extend chosen base, accept property name via `PropertyFilterOptions`
       - Add `readonly filterType = FILTER_TYPE.{Name}Filter as const`
       - Implement `parseValues(targets)` to extract and sort available values from the target entities
       - Set `multipleValues` in super() call if the question/property yields arrays
       - Pattern for question filter: `filter/number/numberQuestionFilter.ts`
       - Pattern for property filter: `filter/enumerated/objectFilter.ts`

    3. **Register in FILTER_TYPE** `filter/base/filterTypes.ts`
       - Add `{Name}Filter: '{name}Filter'` to the `FILTER_TYPE` const object
       - Add `[FILTER_TYPE.{Name}Filter]: {Name}Filter` to `FilterTypeMap` type
       - Import the concrete filter type at the top of the file

    4. **Update type guards** `utils/typeGuards.ts`
       - If new base category: add a new `is{Category}Filter()` function checking `obj.filterType` against the relevant `FILTER_TYPE` values
       - If extending existing base: add `obj.filterType === FILTER_TYPE.{Name}Filter` to the existing category guard (e.g., `isEnumeratedFilter()`, `isNumberFilter()`, `isTextFilter()`)
       - Import the new concrete filter type for the return type annotation

    5. **Add barrel exports** at each directory level:
       - `filter/{category}/index.ts` -- add `export * from './{name}Filter'`
       - `filter/index.ts` -- confirm it already re-exports from `'./{category}'` (if new category, add `export * from './{category}'`)
       - `src/index.ts` -- confirm it already re-exports from `'./filter'` (no change needed)

    6. **Add tests** `tests/filter.test.ts` (relative to `packages/filters/`)
       - Import the new filter class from `../src`
       - Create test entities matching the filter's target type
       - Test constructor with appropriate options (question or property)
       - Test `parseValues()` returns correct values
       - Test `apply()` with include/exclude rules
       - Test `apply()` with MISSING_FILTER_VALUE handling
       - Test that `filterType` equals the expected `FILTER_TYPE.{Name}Filter` value
       - Pattern: follow existing NumberQuestionFilter test block in filter.test.ts

    **Guide 2: Adding a Question-Type Filter Variant** (new XxxQuestionFilter for a new data package question type)

    Reference implementation: `NumberQuestionFilter` (see `filter/number/numberQuestionFilter.ts`).

    This is a simplified version of Guide 1 for when you only need a new concrete filter for a new question type from @openvaa/data, extending an existing base class.

    Cross-reference: If also adding a new question type to @openvaa/data, complete the data skill's extension-patterns.md "Adding a New Question Type" guide first.

    Steps:

    1. **Identify the base class** -- determine which of the 3 base filter categories matches the new question type's answer value:
       - Enumerated values (choices with ids) -> extend `EnumeratedFilter`
       - Numeric values -> extend `NumberFilter`
       - Text/string values -> extend `TextFilter`

    2. **Create concrete filter class** `filter/{category}/{name}QuestionFilter.ts`
       - Extend the identified base class
       - Constructor accepts the new question type via `QuestionFilterOptions<{QuestionType}>`
       - Add `readonly filterType = FILTER_TYPE.{Name}QuestionFilter as const`
       - Auto-detect `multipleValues` if the question type has single/multiple variants (use `isMultipleChoiceQuestion()` or `isObjectType()` pattern)
       - Implement `parseValues(targets)` to extract available values
       - Pattern: `filter/number/numberQuestionFilter.ts` for numeric, `filter/enumerated/choiceQuestionFilter.ts` for enumerated

    3. **Register** -- follow Guide 1 steps 3-5 (FILTER_TYPE, type guards, barrel exports)

    4. **Add tests** -- follow Guide 1 step 6, using `DataRoot` to create the question instance (import from @openvaa/data)

    **Verification After Extension** section:

    1. `cd packages/filters && yarn test:unit` -- all existing tests pass (no regressions)
    2. New test cases pass with expected assertions
    3. `yarn build:shared` -- builds without errors (filters is part of shared build)
    4. FILTER_TYPE const, FilterTypeMap type, and type guards are all in sync (same set of filter types)
    5. Barrel exports chain from subdirectory index.ts -> filter/index.ts -> src/index.ts
    6. New filter's `filterType` property matches the `FILTER_TYPE` const value exactly
