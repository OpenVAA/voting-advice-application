**Step 2: Convert all translation files to inlang message format.**

For each of the 7 locales, for each of the 46 JSON files:
1. Read the source from `apps/frontend/src/lib/i18n/translations/{locale}/{file}.json`
2. Convert to inlang variant format
3. Write to `apps/frontend/messages/{locale}/{file}.json`

**Conversion rules:**

**Simple strings (no ICU patterns):** Copy unchanged. `"hello": "Hello world"` stays as `"hello": "Hello world"`.

**Simple variable interpolation:** Copy unchanged. `"greeting": "Hello {name}"` stays as `"greeting": "Hello {name}"`. Inlang supports `{variable}` syntax directly.

**ICU plural patterns:** Convert to inlang variant syntax. Example:
```json
// BEFORE (ICU):
"numShown": "{numShown, plural, =0 {No candidates} =1 {1 candidate} other {# candidates}}"

// AFTER (inlang variant):
"numShown": [{
  "declarations": ["input numShown", "local numPlural = numShown: plural"],
  "selectors": ["numPlural"],
  "match": {
    "numPlural=zero": "No candidates",
    "numPlural=one": "{numShown} candidate",
    "numPlural=other": "{numShown} candidates"
  }
}]
```

Mapping: ICU `=0` -> `zero`, ICU `=1` -> `one`, ICU `other` -> `other`. The `#` placeholder (ICU count reference) becomes `{variableName}` (explicit variable reference).

**ICU date patterns:** Convert to inlang datetime registry. Example:
```json
// BEFORE (ICU):
"dateInfo": "The election date is {electionDate, date, ::yyyyMMdd}."

// AFTER (inlang variant):
"dateInfo": [{
  "declarations": [
    "input electionDate",
    "local formattedDate = electionDate: datetime year=numeric month=2-digit day=2-digit"
  ],
  "match": {
    "electionDate=*": "The election date is {formattedDate}."
  }
}]
```
