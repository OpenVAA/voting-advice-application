**Step 1: Rewrite app.css**

Replace `apps/frontend/src/app.css` entirely. The new file must follow this exact order:

1. `@import "tailwindcss";` (replaces the three `@tailwind base/components/utilities` directives)
2. `@config "./tailwind.config.mjs";` (loads existing JS config including DaisyUI 4 plugin) -- MUST come AFTER @import
3. `@source inline()` directives (replaces safelist which is NOT supported via @config in TW4)
4. Existing `@layer base { ... }` styles (preserved from current app.css)
5. Existing `@layer components { ... }` styles (preserved from current app.css)
6. Existing `@layer utilities { ... }` styles (preserved from current app.css)

The `@source inline()` directives must cover all classes from the current safelist array. The safelist generates classes for these prefixes: `btn-`, `bg-`, `fill-`, `dark:fill-`, `text-` with these color names: current, primary, secondary, accent, neutral, base-100, base-200, base-300, info, success, warning, error, base-content, primary-content, secondary-content, accent-content, info-content, success-content, warning-content, error-content, white.

Write the @source inline() directives using brace expansion:
```css
@source inline("btn-{current,primary,secondary,accent,neutral,base-100,base-200,base-300,info,success,warning,error,base-content,primary-content,secondary-content,accent-content,info-content,success-content,warning-content,error-content,white}");
@source inline("bg-{current,primary,secondary,accent,neutral,base-100,base-200,base-300,info,success,warning,error,base-content,primary-content,secondary-content,accent-content,info-content,success-content,warning-content,error-content,white}");
@source inline("fill-{current,primary,secondary,accent,neutral,base-100,base-200,base-300,info,success,warning,error,base-content,primary-content,secondary-content,accent-content,info-content,success-content,warning-content,error-content,white}");
@source inline("{dark|}fill-{current,primary,secondary,accent,neutral,base-100,base-200,base-300,info,success,warning,error,base-content,primary-content,secondary-content,accent-content,info-content,success-content,warning-content,error-content,white}");
@source inline("text-{current,primary,secondary,accent,neutral,base-100,base-200,base-300,info,success,warning,error,base-content,primary-content,secondary-content,accent-content,info-content,success-content,warning-content,error-content,white}");
```

Note: The `{dark|}` syntax generates both `fill-X` and `dark:fill-X` variants. If the brace expansion syntax for the dark variant doesn't work, use a separate `@source inline()` line with explicit `dark:fill-` classes instead.

Keep ALL existing `@layer base`, `@layer components`, and `@layer utilities` content exactly as-is. The `@apply` directives in app.css will resolve correctly because the file is the main CSS entry point processed by @tailwindcss/vite.
