<!-- OKLCH Variable Rename Mapping (DaisyUI 4 -> 5) — CRITICAL for this plan -->

IMPORTANT: DaisyUI 5 stores FULL color values in --color-* variables.
  OLD: oklch(var(--p))     -->  NEW: var(--color-primary)         (NOT oklch(var(--color-primary)))
  OLD: oklch(var(--n))     -->  NEW: var(--color-neutral)
  OLD: oklch(var(--b1))    -->  NEW: var(--color-base-100)
  OLD: oklch(var(--b2))    -->  NEW: var(--color-base-200)
  OLD: oklch(var(--b3))    -->  NEW: var(--color-base-300)
  OLD: oklch(var(--bc))    -->  NEW: var(--color-base-content)

IMPORTANT: For inline style variables that pass bare OKLCH components (e.g., var(--b1) without oklch()):
  QuestionChoices.svelte sets: style:--radio-bg={onShadedBg ? 'var(--b2)' : 'var(--b1)'}
  Then uses: bg-[oklch(var(--radio-bg))]

  In DaisyUI 5, --color-base-100 already contains the full color value.
  So the pattern becomes:
    style:--radio-bg={onShadedBg ? 'var(--color-base-200)' : 'var(--color-base-100)'}
  And usage becomes:
    bg-[var(--radio-bg)]     (NO oklch() wrapper — the value is already a full color)

DaisyUI 5 Class Name Changes:
  form-control       --> fieldset (or just remove, use plain HTML structure)
  label-text         --> (removed; style labels directly or use DaisyUI label element)
  label-text-alt     --> (removed)
  input-bordered     --> (removed; inputs have borders by default in DaisyUI 5)
  select-bordered    --> (removed; selects have borders by default in DaisyUI 5)
  textarea-bordered  --> (removed; textareas have borders by default in DaisyUI 5)

DaisyUI 4 Config Variable Renames:
  var(--rounded-box)   --> var(--radius-box)

TW3 Opacity Migration:
  !bg-opacity-30       --> bg-white/30 (used on white-background buttons)
