# Requirements: OpenVAA v1.3 Svelte 5 Migration (Content)

**Defined:** 2026-03-18
**Core Value:** A reliable, well-tested VAA framework that developers can confidently extend, customize, and deploy for real elections.

## v1.3 Requirements

Requirements for voter app content migration to Svelte 5 idioms. Context system rewrite deferred to v1.4.

### Component Migration

- [x] **COMP-01**: All shared and voter-app leaf components use `$props()` instead of `export let`
- [x] **COMP-02**: All `$$restProps`/`$$slots`/`$$Props` removed from migrated components and replaced with runes equivalents
- [x] **COMP-03**: Component-level event forwarding (`on:click` etc.) replaced with callback props across all migrated components
- [x] **COMP-04**: `createEventDispatcher` replaced with callback props in all 6 dispatching components (Alert, Expander, Navigation, SurveyButton, Feedback, DataConsent)
- [x] **COMP-05**: Named `<slot>` elements replaced with `{@render}` snippet props in all container components
- [x] **COMP-06**: `$bindable()` annotations added to all props that use `bind:` (92 occurrences across 40 files)
- [x] **COMP-07**: `svelte:self` replaced with explicit self-import in EntityCard and EntityTag
- [x] **COMP-08**: All `svelte:component` deprecation warnings resolved (7 occurrences, 3 files)
- [x] **COMP-09**: Event modifiers (`|once`, `|preventDefault`, `|capture`) replaced with inline JavaScript equivalents

### Layout Migration

- [x] **LAYOUT-01**: Root Layout.svelte migrated to runes with `$bindable()` props and snippet-based content
- [x] **LAYOUT-02**: MainContent.svelte 6 named slots converted to snippet props with all consumer call sites updated

### Route Migration

- [x] **ROUTE-01**: All voter route `$:` reactive statements converted to `$derived`/`$derived.by()`/`$effect` (correct rune per statement)
- [x] **ROUTE-02**: All voter route `on:event` directives replaced with native event attributes or callback props
- [x] **ROUTE-03**: All voter route `<slot>` usage converted to `{@render}` snippets
- [x] **ROUTE-04**: Root `+layout.svelte` async data-loading pattern migrated from `$:` to `$effect`

### Cleanup

- [x] **CLEAN-01**: All v1.3-scoped TODO[Svelte 5] markers resolved (approximately 10 in voter/shared scope)
- [x] **CLEAN-02**: Candidate app call sites updated with syntax-only changes where shared components changed API (snippet syntax, callback props)

### Validation

- [x] **VAL-01**: All 92 E2E tests pass after migration
- [x] **VAL-02**: TypeScript check passes with zero errors in voter app and shared components
- [x] **VAL-03**: Zero legacy Svelte 4 patterns (`$:`, `on:event` directives, `<slot>`, `$$restProps`, `$$slots`, `$$Props`, `createEventDispatcher`) remaining in voter app routes and shared components

## v1.4+ Requirements

Deferred to future milestones. Tracked but not in current roadmap.

### Context System

- **CTX-01**: Custom store utilities (`parsimoniusDerived`, `storageWritable`, `stackedStore`) replaced with native rune equivalents
- **CTX-02**: `$app/stores` → `$app/state` migration in 3 manual `.ts` files (paramStore, pageDatumStore, authContext)
- **CTX-03**: Context classes (I18n → Component → Data → App → Voter) migrated to `$state`/`$derived` with `createContext<T>` API
- **CTX-04**: `alwaysNotifyStore` eliminated via version-counter signal pattern for DataRoot reactivity

### Candidate App

- **CAND-01**: Candidate app routes migrated to Svelte 5 runes (10 route files + 7 candidate-only components)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Context system rewrite | Highest risk, deepest dependency — separate milestone for coordinated migration |
| `$app/stores` → `$app/state` | Deprecated but functional; tied to context utility rewrite |
| `parsimoniusDerived` elimination | Requires context files as `.svelte.ts` with rune-based inputs |
| `alwaysNotifyStore` removal | Requires context class migration and DataRoot version-counter pattern |
| Candidate app logic migration | Deferred to v1.4; only syntax-only call site updates in v1.3 |
| Admin app routes | Planned for separate rebuild milestone |
