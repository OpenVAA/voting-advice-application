# Requirements: OpenVAA v1.1 Monorepo Refresh

**Defined:** 2026-03-12
**Core Value:** A reliable, well-tested VAA framework that developers can confidently extend, customize, and deploy for real elections.

## v1.1 Requirements

Requirements for the Monorepo Refresh milestone. Each maps to roadmap phases.

### Build Orchestration

- [x] **BUILD-01**: Turborepo is installed and configured with `turbo.json` for all workspace tasks
- [x] **BUILD-02**: Build tasks execute in topological order with dependency-aware caching
- [x] **BUILD-03**: Local build caching skips unchanged packages on rebuild
- [x] **BUILD-04**: Deno compatibility impact of Turborepo is evaluated and documented

### Directory Organization

- [x] **DIR-01**: Frontend, backend (Strapi), and docs are moved to `apps/` directory
- [x] **DIR-02**: Docker Compose configs updated for new paths and dev stack starts correctly
- [x] **DIR-03**: CI workflows updated for new directory structure
- [x] **DIR-04**: E2E tests pass with new directory layout
- [x] **DIR-05**: TypeScript project references updated for new paths

### Version Management

- [x] **VER-01**: Changesets CLI is installed and configured with `.changeset/config.json`
- [x] **VER-02**: Per-package CHANGELOG.md files are generated from changeset descriptions
- [x] **VER-03**: GitHub Action creates automated version PRs on changeset merge
- [ ] **VER-04**: Changeset bot comments on PRs missing changesets
- [x] **VER-05**: npm publishing step exists in pipeline but can be easily disabled/enabled

### Package Publishing

- [x] **PUB-01**: `@openvaa` npm org is registered and configured
- [x] **PUB-02**: core, data, matching, filters have complete npm metadata (license, description, repository, files, publishConfig)
- [x] **PUB-03**: `"private": true` removed from publishable packages
- [x] **PUB-04**: tsup replaces tsc + tsc-esm-fix for publishable package builds (ESM+CJS dual output)
- [x] **PUB-05**: Published packages do not contain `workspace:^` protocol strings
- [x] **PUB-06**: Package installation verified in a fresh Node.js project

### Polish & Optimization

- [x] **POL-01**: Vercel remote cache enabled for CI and local builds
- [x] **POL-02**: Yarn upgraded to 4.10+ with catalogs for centralized dependency versions
- [x] **POL-03**: Lint and typecheck run per-workspace with Turborepo caching

### Bug Fixes

- [x] **FIX-01**: app-shared ESM build typo (`packagec.json`) is fixed

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Package manager migration (pnpm) | High risk, low reward — Turborepo layers on top of Yarn 4 |
| Nx adoption | Overkill for 9-package monorepo, too opinionated |
| Lerna adoption | Legacy tool, effectively an Nx wrapper now |
| Docs site migration to Starlight | Current SvelteKit docs work and have deep monorepo integration |
| Docs split to separate repo | Docs depend on internal packages and generate API docs from source |
| semantic-release | Poor monorepo support, commit-message-driven vs explicit-intent |
| Publishing all packages | Only core/data/matching/filters are general-purpose; others are internal |
