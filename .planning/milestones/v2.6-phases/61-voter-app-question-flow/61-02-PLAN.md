<objective>
Restore category-selection reactivity in the voter app so the question counter updates correctly on every category toggle, and ensure a fresh voter session defaults to all opinion categories checked. Migrate `selectedQuestionCategoryIds` from the `sessionStorageWritable` + `fromStore` bridge to a pure Svelte 5 `$state`, seed the default inside the context via a guarded `$effect`, and simplify the page's `onMount` to only handle the stale-ID filter.

Purpose: Close QUESTION-03 — the Phase 58 UAT "intermittent 0-counter" bug. Per RESEARCH §Pitfall 1, the root cause is `bind:group` on a getter/setter context accessor backed by `fromStore(sessionStorageWritable(...))`; this is the only `bind:group` in the frontend that targets a non-`$state` accessor and it exhibits the exact "sometimes works, sometimes doesn't" symptom cited in the todo. Per D-11, drop sessionStorage entirely — selection is session-only in-memory `$state`.

Output: Rune-native category selection with a guaranteed-all-checked default; `sessionStorage['voterContext-selectedCategoryIds']` persistence removed.
</objective>
