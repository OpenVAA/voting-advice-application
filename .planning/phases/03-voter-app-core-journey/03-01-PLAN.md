**Auto-implication strategy:** The CONTEXT requires "single election + single constituency, both auto-implied." The `(located)` gate auto-implies elections when only 1 election exists, and constituencies when every election has exactly 1 constituency group with 1 constituency. To achieve this, the voter dataset does NOT create a separate election. Instead it reuses the default dataset's single election (`test-election-1`). The default dataset is modified to have only 1 constituency (removing `test-constituency-beta`), giving the single election exactly 1 CG with 1 constituency. This makes auto-implication work: Home -> Intro -> Questions -> Results with no selection pages.

**1c. Create `voter-dataset.json`** with voter-specific test data:

The voter dataset does NOT include elections, constituency groups, or constituencies -- it reuses those from the default dataset. It adds:

- 2 question categories (opinion only): `test-voter-cat-economy` (4 questions), `test-voter-cat-social` (4 questions), both linked to `test-election-1`
- 8 Likert-5 opinion questions: `test-voter-q-1` through `test-voter-q-8`, using existing `test-qt-likert5` question type from default dataset. Questions 1-4 in `test-voter-cat-economy`, questions 5-8 in `test-voter-cat-social`.
- 2 parties: `test-voter-party-a` ("Voter Test Party Alpha"), `test-voter-party-b` ("Voter Test Party Beta")
- 7 candidates with deterministic answer patterns:
  - `test-voter-cand-agree`: all answers = 5 (perfect match when voter answers all 5), `termsOfUseAccepted`
  - `test-voter-cand-close`: answers = [5,5,4,4,5,5,4,4] (close match), `termsOfUseAccepted`
  - `test-voter-cand-neutral`: all answers = 3 (medium match), `termsOfUseAccepted`
  - `test-voter-cand-oppose`: all answers = 1 (worst match), `termsOfUseAccepted`
  - `test-voter-cand-mixed`: answers = [5,1,5,1,5,1,5,1] (alternating), `termsOfUseAccepted`
  - `test-voter-cand-partial`: only answers questions 1-4 (rest missing), `termsOfUseAccepted`
  - `test-voter-cand-hidden`: all answers = 5 but NO `termsOfUseAccepted` (should NOT appear in results)
- 7 nominations: all candidates nominated in `test-election-1` / `test-constituency-alpha`. Party distribution: agree/close/neutral in party-a, oppose/mixed/partial in party-b, hidden in party-a.
- Each nomination has a unique externalId: `test-voter-nom-agree`, `test-voter-nom-close`, etc.

Answer design rationale (voter answers all "5" = Fully agree):
- `cand-agree` = 100% match (distance 0)
- `cand-close` = high match (small distance)
- `cand-neutral` = 50% match
- `cand-oppose` = 0% match (max distance)
- `cand-mixed` = 50% match
- `cand-partial` = computed on 4 of 8 questions, should still appear
- `cand-hidden` = should NOT appear (no termsOfUseAccepted)

**3a. Create `voter.fixture.ts`:**

A parameterizable fixture that answers N Likert questions with a specified value. The fixture navigates through the voter app's question flow, producing a page on the results page with localStorage populated with voter answers.

Because the data setup ensures a single election and single constituency, auto-implication is guaranteed. The fixture navigates: Home -> (auto-implied) -> Intro -> Questions -> Results with no election/constituency selection pages.

```typescript
import { test as base } from '@playwright/test';
import { buildRoute } from '../utils/buildRoute';
import { testIds } from '../utils/testIds';
import type { Page } from '@playwright/test';

type VoterFixtureOptions = {
  /** Number of questions to answer. Default: all (8) */
  voterAnswerCount: number;
  /** Likert answer value index (0-based: 0=Fully disagree, 4=Fully agree). Default: 4 */
  voterAnswerIndex: number;
};

type VoterFixtures = VoterFixtureOptions & {
  /** A page with voter answers already submitted, currently on the results page */
  answeredVoterPage: Page;
};

export const voterTest = base.extend<VoterFixtures>({
  voterAnswerCount: [8, { option: true }],
  voterAnswerIndex: [4, { option: true }],

  answeredVoterPage: async ({ page, voterAnswerCount, voterAnswerIndex }, use) => {
    // Navigate to home page
    await page.goto(buildRoute({ route: 'Home', locale: 'en' }));

    // Click start -- auto-implication handles election + constituency (single of each)
    await page.getByTestId(testIds.voter.home.startButton).click();

    // Should land on intro page (no election/constituency selection)
    await page.getByTestId(testIds.voter.intro.startButton).click();

    // Answer questions
    for (let i = 0; i < voterAnswerCount; i++) {
      // Click the answer option (Likert choice at the specified index)
      await page.getByTestId(testIds.voter.questions.answerOption).nth(voterAnswerIndex).click();

      // Wait for auto-advance to next question or results
      if (i < voterAnswerCount - 1) {
        await page.waitForURL(/\/questions\//);
      }
    }

    // After answering last question, click "Results" button
    await page.getByTestId(testIds.voter.questions.nextButton).click();

    // Should now be on results page
    await page.getByTestId(testIds.voter.results.list).waitFor({ state: 'visible', timeout: 10000 });

    await use(page);
  }
});
```

Note: The exact flow may need adjustment based on auto-advance behavior. The key is that after answering the last question, the next button shows "Results" and clicking it navigates to results. If auto-advance skips ahead automatically, remove the explicit next click and just wait for results.
