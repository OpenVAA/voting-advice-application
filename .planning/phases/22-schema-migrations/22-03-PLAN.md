The `questions` table requires choices for singleChoiceOrdinal answers. For upsert_answers tests,
use a simpler test: verify the UPDATE goes through RLS rather than testing the full answer validation.
Use a text-type question or set answers to '{}' to bypass validate_answers_jsonb.
Actually: to test upsert_answers without triggering validate_answers_jsonb(), call with an empty
answers jsonb '{}' or with a valid question ID. The simplest approach:
1. Add a question to the test data with known ID in create_test_data (already done for question_a)
2. Use answers that match question_a's type (singleChoiceOrdinal with choices)
OR: Test the RLS access control directly using a raw UPDATE statement, then test the RPC with empty answers.
Recommended: test upsert_answers with empty answers '{}' to verify it runs and returns '{}' without
triggering validation. Separately test that candidate_b cannot call it for candidate_a's row.

For upsert_answers merge behavior with actual values, read apps/supabase/supabase/tests/database/08-triggers.test.sql
to see how the existing trigger tests set up valid answers for singleChoiceOrdinal questions.
