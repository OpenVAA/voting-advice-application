```typescript
  // _getConstituencyData, _getNominationData, _getEntityData, _getQuestionData

```

All 7 methods use PostgREST query builder via `this.supabase`, localize data via `getLocalized`/`toDataObject`, convert storage paths via `parseStoredImage`, and parse answers via `parseAnswers`.

Key implementation details:
- `_getAppSettings`: queries `app_settings` table, returns `settings` JSONB column, localizes notification title/content
- `_getAppCustomization`: queries `app_settings.customization` JSONB, localizes publisherName/translationOverrides/FAQ, converts image paths
- `_getElectionData`: queries `elections` with `election_constituency_groups` join, maps `election_date`/`current_round`/`election_type`
- `_getConstituencyData`: two queries — `constituency_groups` with join + `constituencies`, localizes keywords
- `_getNominationData`: calls `get_nominations` RPC, deduplicates entities via Map, sets candidate-specific fields
- `_getEntityData`: queries `candidates` and/or `organizations` tables based on entityType filter
- `_getQuestionData`: queries `question_categories` + `questions`, localizes choice labels, filters by electionId
