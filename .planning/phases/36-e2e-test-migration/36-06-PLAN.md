<title>Diff-merge candidate-registration.spec.ts for Supabase auth flow</title>
<read_first>
- `tests/tests/specs/candidate/candidate-registration.spec.ts` (current — Strapi auth flow)
- `git show feat-gsd-supabase-migration:tests/tests/specs/candidate/candidate-registration.spec.ts` (parallel — Supabase auth flow)
- `tests/tests/utils/emailHelper.ts` (after Plan 03 — must have toCallbackUrl)
- `tests/tests/utils/supabaseAdminClient.ts` (after Plan 01 — method signatures)
</read_first>
<action>
This is the most complex diff-merge. The auth flow is fundamentally different:

**Registration flow changes (Strapi → Supabase):**
- Strapi: sendEmail with candidateId + requireRegistrationKey → user clicks link → set password → redirect to login → manual login → ToU acceptance via updateCandidate
- Supabase: sendEmail with candidateExternalId + email → user clicks transformed callback URL → set password → auto-login (session from verifyOtp) → ToU acceptance via UI

**Password reset flow changes:**
- Strapi: sendForgotPassword({ documentId }) → click raw link → set password → redirect to login → manual login → restore via setPassword({ documentId, password })
- Supabase: sendForgotPassword(email) → click transformed callback URL → set password → auto-login → restore via setPassword(email, password)

**Changes to apply:**

1. **Imports:** Replace `StrapiAdminClient` with `SupabaseAdminClient`. Add `toCallbackUrl` to emailHelper imports. Remove `buildRoute` import if unused. Remove `testIds` import if unused after ToU flow changes.

2. **Remove lifecycle:** Remove `test.beforeAll` (client.login) and `test.afterAll` (client.dispose) blocks.

3. **Registration — send email test:**
   - Replace `client.findData` + `client.sendEmail({ candidateId: documentId!, ... })` with:
     ```typescript
     await client.sendEmail({
       candidateExternalId,
       email: candidateEmail,
       subject: 'Registration',
       content: 'Click here to register: {LINK}'
     });
     ```
   - Add `const candidateExternalId = candidateAddendum.candidates[0].externalId;` to shared state
   - Extract link and transform: `registrationLink = toCallbackUrl(rawLink!);`

4. **Registration — complete registration test:**
   - Keep `test.setTimeout(60000)` (parallel branch also uses 60s)
   - Use `page.getByTestId('register-password')` / `page.getByTestId('register-confirm-password')` / `page.getByTestId('register-password-submit')` for password fields (keep testIds from current if they match, or use string literals from parallel branch — check what testIds.candidate.register contains)
   - Password fill: Use `getByTestId('password-field')` within wrapper (parallel branch pattern)
   - After registration: expect redirect to candidate home (NOT login page) — Supabase invite creates authenticated session:
     ```typescript
     await expect(page).toHaveURL(/\/candidate(?!.*login)/, { timeout: 10000 });
     ```
   - ToU acceptance via UI (not API):
     ```typescript
     const touCheckbox = page.getByTestId('terms-checkbox');
     await expect(touCheckbox).toBeVisible({ timeout: 10000 });
     await touCheckbox.check();
     const continueButton = page.getByRole('button', { name: /continue/i });
     await expect(continueButton).toBeEnabled({ timeout: 10000 });
     await continueButton.click();
     ```
   - Verify home page: `await expect(page.getByTestId('candidate-home-status')).toBeVisible({ timeout: 15000 });`
   - Remove the entire Strapi cookie workaround block (allCookies, addCookies, about:blank navigation)
   - Remove `client.findData` + `client.updateCandidate` for ToU acceptance

5. **Password reset test:**
   - Remove `test.beforeAll`/`test.afterAll` for client lifecycle
   - Replace `client.findData` + `client.sendForgotPassword({ documentId })` with:
     ```typescript
     await client.sendForgotPassword(candidateEmail);
     ```
   - Transform reset link: `const resetLink = toCallbackUrl(rawResetLink!);` then `await page.goto(resetLink);`
   - Use `page.getByTestId('password-field')` for password inputs (parallel branch pattern)
   - Use `page.getByTestId('password-reset-submit')` for submit
   - After reset: expect auto-login to candidate home (NOT login page):
     ```typescript
     await expect(page).toHaveURL(/\/candidate(?!.*login|.*password)/, { timeout: 10000 });
     await expect(page.getByTestId('candidate-home-status')).toBeVisible();
     ```
   - Restore password: `await client.setPassword(candidateEmail, originalPassword);` (email-based, not documentId-based)
   - Remove the manual login verification step (Steps 7-8 in current) — Supabase auto-authenticates after recovery

6. **Keep from current branch:** `import { expect, test } from '../../fixtures'` (not `@playwright/test`)
</action>
<acceptance_criteria>

- Registration uses `client.sendEmail({ candidateExternalId, email, ... })` (not `candidateId`)
- Registration link transformed via `toCallbackUrl(rawLink!)`
- After registration, expects URL matching `/candidate` (not `/login`)
- ToU accepted via UI checkbox (not API updateCandidate)
