<action>
Replace the contents of `tests/tests/utils/emailHelper.ts` with the Mailpit version from the parallel branch. The new version:

1. **Base URL:** `MAILPIT_URL = process.env.INBUCKET_URL ?? 'http://localhost:54324'`

2. **Interfaces:** Replace `SESEmail`/`SESMailbox` with `MailpitMessageSummary`/`MailpitMessage`:
   ```typescript
   interface MailpitMessageSummary {
     ID: string;
     MessageID: string;
     From: { Name: string; Address: string };
     To: Array<{ Name: string; Address: string }>;
     Subject: string;
     Created: string;
     Size: number;
     Read: boolean;
   }

   interface MailpitMessage extends MailpitMessageSummary {
     HTML: string;
     Text: string;
   }
   ```

3. **Functions to include:**
   - `fetchEmails(recipientEmail: string)` — searches via `GET /api/v1/search?query=to:{email}`
   - `getLatestEmailHtml(recipientEmail, skipCount)` — fetches full message via `GET /api/v1/message/{ID}`
   - `extractLinkFromHtml(html)` — unchanged (still uses cheerio)
   - `getRegistrationLink(recipientEmail)` — combines fetch + extract, error message references Mailpit
   - `toCallbackUrl(verifyLink, callbackPath)` — NEW: transforms Supabase Auth verify link into direct frontend auth callback URL
   - `countEmailsForRecipient(recipientEmail)` — counts via fetchEmails
   - `purgeMailbox(_recipientEmail)` — NEW: deletes all messages via `DELETE /api/v1/messages`

4. **Removed imports:** Drop `import { request } from '@playwright/test'` and `import { simpleParser } from 'mailparser'`
5. **Kept imports:** `import { load } from 'cheerio'` (still used for link extraction)
6. **Use native `fetch`** instead of Playwright's `request` API

Copy the complete file from `git show feat-gsd-supabase-migration:tests/tests/utils/emailHelper.ts`.
</action>
