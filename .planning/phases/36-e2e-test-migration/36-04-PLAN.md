- `tests/tests/utils/supabaseAdminClient.ts` (new admin client API)
</read_first>
<action>
Replace the contents of `data.setup.ts` with the Supabase version. Key changes from current:

1. **Import:** Replace `StrapiAdminClient` with `SupabaseAdminClient`:
   ```typescript
   import { SupabaseAdminClient } from '../utils/supabaseAdminClient';
   ```

2. **Remove lifecycle:** Remove `await client.login()` and `await client.dispose()`. Constructor is sufficient.

3. **Remove uploadTestAssets:** Remove `import { uploadTestAssets } from '../utils/uploadTestAssets'` and `await uploadTestAssets()` call. Supabase Storage handles test assets differently.

4. **Delete operation:** Replace `client.deleteData({ collection: prefix })` with `client.bulkDelete({ collection: { prefix: prefix } })`:
   ```typescript
   const deleteResult = await client.bulkDelete({
     nominations: { prefix: TEST_DATA_PREFIX },
     candidates: { prefix: TEST_DATA_PREFIX },
     questions: { prefix: TEST_DATA_PREFIX },
     question_categories: { prefix: TEST_DATA_PREFIX },
     organizations: { prefix: TEST_DATA_PREFIX },
     constituency_groups: { prefix: TEST_DATA_PREFIX },
     constituencies: { prefix: TEST_DATA_PREFIX },
     elections: { prefix: TEST_DATA_PREFIX }
   });
   ```
   Note: collections use snake_case names, candidates ARE deleted (unlike Strapi), and `alliances`/`questionTypes` are removed.

5. **Import operations:** Replace single `client.importData(data)` with three-step:
   ```typescript
   await client.bulkImport(data as Record<string, unknown[]>);
   await client.importAnswers(data as Record<string, unknown[]>);
   await client.linkJoinTables(data as Record<string, unknown[]>);
   ```
   Apply to default dataset, voter dataset, and candidate addendum (no importAnswers for addendum since unregistered candidates have no answers).

6. **App settings:** The `updateAppSettings` call now does deep merge (not replace). Add `results` section:
   ```typescript
   results: {
     cardContents: {
       candidate: ['submatches'],
       organization: ['candidates']
     },
     sections: ['candidate', 'organization']
   },
   ```

7. **Password restoration:** Replace the `findData` + `setPassword({ documentId })` + `forceRegister({ documentId })` pattern with:
   ```typescript
   await client.unregisterCandidate('mock.candidate.2@openvaa.org');
   await client.forceRegister('test-candidate-alpha', 'mock.candidate.2@openvaa.org', TEST_CANDIDATE_PASSWORD);
   ```

8. **Remove `TEST_CANDIDATE_PASSWORD` import** if it was only used for the findData+setPassword pattern above — but check: it IS still used for forceRegister. Keep the import.

Use `git show feat-gsd-supabase-migration:tests/tests/setup/data.setup.ts` as the reference implementation.
</action>
