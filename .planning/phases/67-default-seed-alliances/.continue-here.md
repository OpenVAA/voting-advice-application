## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Settings shallow-merge asymmetry | Server-side `merge_jsonb_column` deep-merges JSONB; client-side `mergeAppSettings` (apps/frontend/src/lib/utils/settings.ts) is shallow `Object.assign` at root keys. Plan 67-01 assumed symmetry and wrote a partial `results` override, which the client merge then used to REPLACE the entire `results` block — wiping `cardContents`, `showFeedbackPopup`, `showSurveyPopup` defaults. Crashed the results page. | advisory | When writing dynamic settings overrides in `packages/dev-seed/`, mirror the FULL TS-default shape from `packages/app-shared/src/settings/dynamicSettings.ts`. Only diff leaf values; never write a partial sub-object at depth ≥ 1. |
| Vite dep-optimization cache stale after package rebuild | After `@openvaa/*` package rebuild, Vite's `node_modules/.vite/` cache holds the OLD bundled dist. Restarting `yarn dev` does NOT invalidate it. Symptom: identical crash persists despite the fix being in `dist/index.js`. | advisory | After rebuilding any `@openvaa/*` package consumed by `apps/frontend`, run `rm -rf apps/frontend/node_modules/.vite` before restarting `yarn dev`. Worth promoting to CLAUDE.md if not already documented. |
| Playwright parity-gate DB contamination | Running `yarn dev:reset-with-data` (default template) BEFORE Playwright contaminates the DB because `tests/tests/setup/data.setup.ts` only clears `test-` prefix rows, not `seed-`. Both datasets coexist → 40 questions instead of 20 → 20 voter-app tests timeout at q18/40 → false-positive parity FAIL of 30p/21f/51c. | advisory | Phase 66 66-VALIDATION.md:73-82 protocol: run `yarn supabase:reset` (clean DB) BEFORE Playwright. Also documented in 67-VERIFICATION.md. Consider promoting to CLAUDE.md or adding a guard in the parity-gate script. |
