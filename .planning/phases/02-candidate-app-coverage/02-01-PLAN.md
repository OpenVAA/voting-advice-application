**1. Create `tests/tests/utils/emailHelper.ts`** following the research pattern exactly:
- Import `load` from `cheerio`, `simpleParser` from `mailparser`, `request` from `@playwright/test`
- Define `SESEmail` interface: `{ Id, Region, Source, RawData, Timestamp }` (all strings)
- Define `SESMailbox` interface: `{ messages: Array<SESEmail> }`
- `SES_INBOX_URL` = `${process.env.LOCALSTACK_ENDPOINT ?? 'http://localhost:4566'}/_aws/ses`
- Export `fetchEmails()`: creates request context, fetches SES_INBOX_URL, parses JSON, disposes context, returns messages array
- Export `getLatestEmailHtml(recipientEmail)`: iterates emails in reverse, parses each with `simpleParser`, checks `to.value` addresses for match (case-insensitive), falls back to checking `RawData` for recipient string, returns `parsed.textAsHtml ?? parsed.html ?? undefined`
- Export `extractLinkFromHtml(html)`: uses cheerio `load(html)`, returns `$('a').first().attr('href') ?? undefined`
- Export `getRegistrationLink(recipientEmail)`: combines the above, throws descriptive errors if no email found or no link found

**2. Extend `tests/tests/utils/strapiAdminClient.ts`** with four new methods:
- `async updateAppSettings(data: Record<string, unknown>): Promise<void>` - PUTs to `/content-manager/single-types/api::app-setting.app-setting` with admin headers. Throws descriptive error on non-OK response. IMPORTANT: Use `data` parameter directly (NOT JSON.stringify) because the content-manager admin API expects parsed JSON, unlike the Admin Tools controllers which do `JSON.parse(ctx.request.body)`.
- `async sendEmail(params: { candidateId: string; subject: string; content: string; requireRegistrationKey?: boolean }): Promise<void>` - POSTs to `/openvaa-admin-tools/send-email` with `data: JSON.stringify(params)` (following existing Admin Tools pattern). Throws on non-OK.
- `async sendForgotPassword(params: { documentId: string }): Promise<void>` - POSTs to `/openvaa-admin-tools/candidate-auth/forgot-password` with `data: JSON.stringify(params)`. NOTE: This method does NOT return the resetUrl. Per user decision, the reset link must be read from SES email, not from the API response. The method only triggers the email sending. Throws on non-OK.
- `async setPassword(params: { documentId: string; password: string }): Promise<void>` - POSTs to `/openvaa-admin-tools/candidate-auth/set-password` with `data: JSON.stringify(params)`. Used to restore candidate passwords after password change/reset tests. Throws on non-OK.
- Keep `requestContext` private but the new methods use `this.requestContext!` and `this.headers` like existing methods.
