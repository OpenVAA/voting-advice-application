### Rationale for not picking path (b1)

Cloning `data.setup.ts` → `data-setup-likert-only.setup.ts` and splitting the `voter-app` Playwright project into `voter-app` + `voter-app-likert-only` is the more orthodox structural fix (mirrors Phase 85-03's variant-multi-election approach), but RESEARCH §3.4 surfaces the unresolved conflict: the 5 directional-asserting PASS_LOCKED tests live in the SAME `voter-app` project as the fixture-fragile tests. Medium blast radius. Path (b2) is the smaller, safer fix.

### Why path (b2) is correct

The 85-04 cluster #2 root cause IS fixture-fragility: the `answeredVoterPage` post-Likert Skip-Next tail loop was sized for sorts 17 + 18 + 1 headroom (cap 3), but sort 19 (number) is also unanswered after the 16-Likert main loop. - Contains the change to ONE file (`tests/tests/fixtures/voter.fixture.ts`)
- Does NOT modify the e2e seed template
- Preserves all 5 PASS_LOCKED tests requiring `test-question-directional-1` by construction
- 3-step headroom (cap = 6 vs. RESEARCH-recommended cap = 5) protects against future non-Likert opinion questions added at sort ≤ 24
