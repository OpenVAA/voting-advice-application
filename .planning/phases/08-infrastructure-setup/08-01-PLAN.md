1. Create the `apps/supabase/` directory.

2. Create `apps/supabase/package.json` with:
   - name: `@openvaa/supabase`
   - private: true
   - version: `0.1.0`
   - scripts:
     - `start`: `supabase start`
     - `stop`: `supabase stop`
     - `reset`: `supabase db reset`
     - `status`: `supabase status`
   - devDependencies: `supabase` (use `^2.78.1` or latest)

3. Update root `package.json`:
   - Add `"apps/*"` to the `workspaces` array (keep existing entries intact)
   - Add alias scripts:
     - `"supabase:start"`: `"yarn workspace @openvaa/supabase start"`
     - `"supabase:stop"`: `"yarn workspace @openvaa/supabase stop"`
     - `"supabase:reset"`: `"yarn workspace @openvaa/supabase reset"`
     - `"supabase:status"`: `"yarn workspace @openvaa/supabase status"`

4. Run `yarn install` to register the new workspace and install supabase CLI.
   - If postinstall fails, retry with `NODE_OPTIONS=--no-experimental-fetch yarn install`.

5. From within `apps/supabase/`, run `supabase init` to create the `supabase/` subdirectory with default config.toml.
   - This creates `apps/supabase/supabase/config.toml`.

6. Edit `apps/supabase/supabase/config.toml` to configure local development:
   - `[project]` id = `"openvaa-local"`
   - `[api]` port = 54321, schemas = `["public", "graphql_public"]`, extra_search_path = `["public", "extensions"]`, max_rows = 1000
   - `[db]` port = 54322, shadow_port = 54320, major_version = 15
   - `[db.seed]` enabled = true, sql_paths = `["./seed.sql"]`
   - `[studio]` enabled = true, port = 54323
   - `[inbucket]` enabled = true, port = 54324, smtp_port = 54325, pop3_port = 54326
   - `[storage]` enabled = true, file_size_limit = `"50MiB"`
   - `[auth]` enabled = true, site_url = `"http://127.0.0.1:5173"`, jwt_expiry = 3600, enable_signup = true, enable_anonymous_sign_ins = false
   - `[auth.email]` enable_signup = true, double_confirm_changes = true, enable_confirmations = false
   - `[edge_runtime]` enabled = true, policy = `"oneshot"`
   - Keep other defaults from `supabase init` output -- only override the values listed above.
