```svelte
<script lang="ts">
  import { onMount } from 'svelte';
  import { getAdminContext } from '$lib/contexts/admin';
  import type { Snippet } from 'svelte';

  let { children }: { children: Snippet } = $props();

  const {
    jobs: { startPolling, stopPolling }
  } = getAdminContext();

  onMount(() => {
    startPolling();
    return () => stopPolling();
  });
</script>

{@render children?.()}
```

Note: `onDestroy` import is REPLACED by `onMount` (we use the cleanup-return form, which fires at unmount). `onDestroy(() => stopPolling())` is removed entirely. If a different site requires SSR-needed data, Pattern 4 fallback is `+page.ts` / `+layout.ts` `load({ fetch })`.
