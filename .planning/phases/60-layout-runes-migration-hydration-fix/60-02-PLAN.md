  if (!isValidResult(data.appCustomizationData, { allowEmpty: true }))

This plan refactors a client-side Svelte component. The trust boundary it touches is **browser ↔ already-resolved loader data** — data arrives via `+layout.ts` / `+layout.server.ts` which run under SvelteKit SSR. No new trust boundary is crossed. The refactor changes WHEN computations run (synchronous `$derived` vs microtask-delayed `.then()`) but not WHAT is computed or what data flows where.
