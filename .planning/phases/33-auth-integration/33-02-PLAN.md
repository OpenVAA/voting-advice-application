# Plan 33-02: Consumer Updates for isAuthenticated

<objective>
Update all consumers of the old `authToken` property from AuthContext to use the new `isAuthenticated` property. This includes candidateContext, candidateUserDataStore, CandidateNav, AdminNav, and adminContext. The candidateUserDataStore switches to the empty-string-authToken pattern for save/reload operations.
</objective>

<must_haves>
- candidateContext destructures `isAuthenticated` from authContext (not `authToken`)
- candidateUserDataStore uses empty string `''` for authToken in save/reload (not `get(authToken)`)
- CandidateNav uses `isAuthenticated` for conditional rendering (not `authToken`)
- AdminNav uses `isAuthenticated` for conditional rendering (not `authToken`)
- adminContext uses empty string `''` in injectAuthToken helper
- auth/index.ts still exports getUserData (authToken.ts export kept for other consumers until Phase 38)
</must_haves>
