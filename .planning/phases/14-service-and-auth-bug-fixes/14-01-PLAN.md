    **Fix 1 — bulk_import ON CONFLICT (016-bulk-operations.sql):**
    In the `_bulk_upsert_record` function, change the format string on line 197 from:
    ```
    ON CONFLICT (project_id, external_id) DO UPDATE SET
    ```
    to:
    ```
    ON CONFLICT (project_id, external_id) WHERE external_id IS NOT NULL DO UPDATE SET
    ```
    This makes the ON CONFLICT clause match the partial unique indexes defined in 015-external-id.sql (all of which have `WHERE external_id IS NOT NULL`). Also update the comment on line 195 to say "partial unique index" instead of "composite unique index".

    **Fix 2 — storage cleanup search_path (014-storage.sql):**
    In `cleanup_entity_storage_files()` (lines 401-402), change:
    ```
    PERFORM delete_storage_object(...)
    ```
    to:
    ```
    PERFORM public.delete_storage_object(...)
    ```
    for both calls (public-assets and private-assets).

    In `cleanup_old_image_file()` (lines 482 and 490), change both:
    ```
    PERFORM delete_storage_object(...)
    ```
    to:
    ```
    PERFORM public.delete_storage_object(...)
    ```
    This is 4 call sites total across the 2 functions. Both functions use `SET search_path = ''` so unqualified calls fail to resolve.
