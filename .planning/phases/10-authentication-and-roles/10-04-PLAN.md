  <action>
Create the Edge Function at `apps/supabase/supabase/functions/invite-candidate/index.ts`.

The function handles POST requests with the following flow:

1. **Parse request** -- accept JSON body with: `firstName` (string, required), `lastName` (string, required), `email` (string, required), `projectId` (string/uuid, required), `organizationId` (string/uuid, optional).

2. **Verify caller is admin** -- create a Supabase client using the caller's Authorization header (anon key + auth token). Call `getUser()` to get the caller's identity. Then verify the caller has an admin role (project_admin, account_admin, or super_admin) for the target project. Do this by decoding the JWT from the Authorization header and checking `user_roles` claims (not a DB lookup). If not authorized, return 403.

3. **Create candidate record** -- using a `service_role` admin client, insert into `candidates` table with `first_name`, `last_name`, `project_id`, and optionally `organization_id`. Select the created record to get its ID.

4. **Send invite email** -- call `supabaseAdmin.auth.admin.inviteUserByEmail(email, { data: { candidate_id: candidate.id, project_id: projectId }, redirectTo: '${SITE_URL}/candidate/complete-registration' })`. Use `Deno.env.get('SUPABASE_URL')` as fallback if SITE_URL is not set. The `data` object goes into `raw_user_meta_data` on the auth user.

5. **Create role assignment** -- insert into `user_roles` with `user_id` from the invite response, `role: 'candidate'`, `scope_type: 'candidate'`, `scope_id: candidate.id`.

6. **Link auth user to candidate** -- update the candidate record to set `auth_user_id` to the invited user's ID.

7. **Return response** -- 201 with `{ success: true, candidateId: candidate.id, userId: inviteData.user.id }`.

**Error handling:**
- 400 for missing required fields
- 403 for non-admin callers
- 500 for database/invite errors, with rollback: if the invite fails after candidate creation, delete the candidate record. If role assignment fails after invite, log but don't fail (invite email already sent).

**Deno imports:**

**Admin role verification approach:**
```typescript
// Decode JWT to check roles from claims (Custom Access Token Hook)
const token = authHeader.replace('Bearer ', '');
const payload = JSON.parse(atob(token.split('.')[1]));
const userRoles = payload.user_roles || [];
const isAdmin = userRoles.some((r: {role: string, scope_type: string, scope_id: string}) =>
  r.role === 'super_admin' ||
  (r.role === 'account_admin') ||
  (r.role === 'project_admin' && r.scope_type === 'project' && r.scope_id === projectId)
);
```
Note: For a more secure check, also call `getUser()` on the caller's client to verify the token is valid server-side, not just locally decoded.
