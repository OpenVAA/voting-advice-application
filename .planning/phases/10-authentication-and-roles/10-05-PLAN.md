Create the Edge Function at `apps/supabase/supabase/functions/signicat-callback/index.ts`.

The function handles POST requests with the following flow:

1. **Parse request** -- accept JSON body with: `id_token` (string, the JWE-encrypted id_token from Signicat OIDC callback) and optionally `project_id` (string/uuid, defaults to DEFAULT_PROJECT_ID env var or the seed default project UUID).

2. **Decrypt JWE token** -- using jose library (Deno import: `https://deno.land/x/jose@v5.9.6/index.ts`):
```typescript
const privateJWKSet = JSON.parse(Deno.env.get('SIGNICAT_DECRYPTION_JWKS')!);
const header = jose.decodeProtectedHeader(id_token);
const privateKey = privateJWKSet.find((jwk: jose.JWK) => jwk.kid === header.kid);
if (!privateKey) throw new Error('No matching decryption key found');
const { plaintext } = await jose.compactDecrypt(
  id_token,
  await jose.importJWK(privateKey, header.alg || 'RSA-OAEP')
);
const innerJwt = new TextDecoder().decode(plaintext);
```

3. **Verify inner JWT** -- verify the decrypted JWT against Signicat's public JWKS:
```typescript
const jwksUri = Deno.env.get('SIGNICAT_JWKS_URI')!;
const { payload } = await jose.jwtVerify(
  innerJwt,
  jose.createRemoteJWKSet(new URL(jwksUri)),
  { audience: Deno.env.get('SIGNICAT_CLIENT_ID') }
);
```

4. **Extract identity claims** -- get `given_name`, `family_name`, `birthdate` from payload. The birthdate serves as the unique identifier for Finnish bank auth (national ID equivalent).

5. **Find or create auth user** -- using service_role admin client:
- Search for existing user by `app_metadata.birthdate_id` match: `supabaseAdmin.auth.admin.listUsers()` and filter. If found, use existing user.
- If not found, create new user: `supabaseAdmin.auth.admin.createUser({ email_confirm: true, app_metadata: { birthdate_id: birthdate, provider: 'signicat' }, user_metadata: { given_name, family_name } })`. Note: email is NOT set at creation time -- per user decision, the candidate is prompted to enter an email after bank auth login.

6. **Find or create candidate record** -- check if a candidate exists linked to this auth user. If not, create one:
```typescript
const { data: existingCandidate } = await supabaseAdmin
  .from('candidates')
  .select('id')
  .eq('auth_user_id', userId)
  .maybeSingle();

if (!existingCandidate) {
  const { data: candidate } = await supabaseAdmin
    .from('candidates')
    .insert({
      first_name: given_name,
      last_name: family_name,
      project_id: projectId,
      auth_user_id: userId
    })
    .select('id')
    .single();

  // Create role assignment
  await supabaseAdmin.from('user_roles').insert({
    user_id: userId,
    role: 'candidate',
    scope_type: 'candidate',
    scope_id: candidate.id
  });
}
```

7. **Generate session** -- create a magic link for immediate login:
```typescript
const { data: linkData } = await supabaseAdmin.auth.admin.generateLink({
  type: 'magiclink',
  email: userId + '@bank-auth.placeholder', // placeholder since no email yet
  options: { redirectTo: siteUrl + '/candidate' }
});
```
Alternative approach if generateLink requires a real email: use `supabaseAdmin.auth.admin.updateUserById(userId, { email_confirm: true })` and return the user info + a session token. The frontend can then use `supabase.auth.setSession()` with the returned tokens.

Return 200 with the session data or magic link URL. The frontend redirects the user to complete their profile (add email).

**Error handling:**
- 400 for missing id_token
- 401 for JWE decryption failure or JWT verification failure (invalid/expired token)
- 500 for database/auth errors

**CORS headers:** Same pattern as invite-candidate.

**Important notes:**
- This Edge Function handles the JWE (encrypted) token path since the existing OpenVAA Signicat integration uses encrypted tokens. If Signicat is configured for plain JWTs, the decryption step is skipped (detect by checking if the token has 5 parts vs 3 parts in the compact serialization).
- The function should gracefully handle both JWE (5-part) and plain JWT (3-part) tokens to support configuration changes.
- Per user decision: after bank auth, the candidate is prompted to enter an email for verification. The candidate can also set a password for email/password login. This is handled by the frontend after the session is established, not by this Edge Function.
