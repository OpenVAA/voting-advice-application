**Part A: Create 017-email-helpers.sql**

Create a `resolve_email_variables(user_ids uuid[], template_body text, template_subject text)` Postgres function that:

1. For each user_id, looks up their entity context:
   - Join user_roles to find the user's role assignments
   - For `candidate` role: join to candidates table to get first_name, last_name, organization_id
   - For `candidate` role with nominations: join nominations -> constituencies -> elections for constituency and election context
   - For `party` role: join to organizations table
   - Resolve localized fields using get_localized() with the user's preferred_locale from auth.users.raw_user_meta_data

2. Returns a table of (user_id uuid, email text, preferred_locale text, variables jsonb) where variables is a flat JSONB object:
   ```json
   {
     "candidate.first_name": "Matti",
     "candidate.last_name": "Meikalainen",
     "organization.name": "SDP",
     "nomination.constituency.name": "Helsinki"
   }
   ```

3. Language: plpgsql, SECURITY DEFINER with SET search_path = '' (needs to read auth.users which is not accessible to regular authenticated users).

4. Grant EXECUTE to authenticated (admin-only in practice since the Edge Function checks admin role before calling).

The function resolves variables for the most common template paths. Unknown variable paths are left as-is (not replaced). The function supports:
- `candidate.first_name`, `candidate.last_name`
- `organization.name` (localized)
- `nomination.constituency.name` (localized)
- `nomination.election.name` (localized)

For each user, the function:
- Gets preferred_locale from `auth.users.raw_user_meta_data->>'preferred_locale'` (default 'en')
- Gets email from `auth.users.email`
- Gets the first `candidate` or `party` role from user_roles
- Based on role, joins to the appropriate entity tables
- Resolves localized fields using get_localized(field, preferred_locale)

**Part B: Create send-email/index.ts**

Follow the established Edge Function pattern from invite-candidate/index.ts:

```typescript
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';
import nodemailer from 'npm:nodemailer@6.9.10';
```

1. **CORS + POST-only** (same pattern as invite-candidate)

2. **Parse request body:**
   ```typescript
   interface SendEmailRequest {
     templates: Record<string, { subject: string; body: string }>;
     recipient_user_ids: string[];
     from?: string;
     dry_run?: boolean;
   }
   ```
   Validate: templates must have at least one locale entry, recipient_user_ids must be non-empty array.

3. **Verify caller is admin:**
   Same pattern as invite-candidate -- getUser() for server-side validation, JWT decode for role claims. Check that caller has project_admin, account_admin, or super_admin role.

4. **Resolve template variables:**
   Create service_role client. Call the `resolve_email_variables` RPC:
   ```typescript
   const { data: recipients, error } = await supabaseAdmin.rpc('resolve_email_variables', {
     user_ids: recipientUserIds,
     template_body: '', // Not used for resolution, just for the type
     template_subject: ''
   });
   ```
   Actually, simplify: the RPC returns user context (email, preferred_locale, variables). The Edge Function does the actual template string replacement.

5. **For each recipient:**
   - Select the template matching their preferred_locale (fall back to first available locale)
   - Replace `{{variable.path}}` placeholders in subject and body with values from the resolved variables
   - Use a simple regex replace: `template.replace(/\{\{(\w+(?:\.\w+)*)\}\}/g, (match, key) => variables[key] ?? match)`

6. **Send or dry-run:**
   - If `dry_run === true`: collect rendered (to, subject, body) per recipient and return as JSON response without sending
   - If `dry_run === false`: create nodemailer transport and send each email

7. **SMTP transport configuration:**
   ```typescript
   const transport = nodemailer.createTransport({
     host: Deno.env.get('SMTP_HOST') || 'inbucket',
     port: parseInt(Deno.env.get('SMTP_PORT') || '2500'),
     secure: false,
     tls: { rejectUnauthorized: false }
   });
   ```
   Local dev: connects to Inbucket via Docker internal network (host: 'inbucket', port: 2500).
   Production: configurable via SMTP_HOST/SMTP_PORT/SMTP_USER/SMTP_PASS env vars.

8. **Return response:**
   ```json
   {
     "success": true,
     "sent": 2,
     "failed": 0,
     "dry_run": false,
     "results": [
       {"user_id": "...", "email": "...", "status": "sent"},
       {"user_id": "...", "email": "...", "status": "sent"}
     ]
   }
   ```
   For dry_run:
   ```json
   {
     "success": true,
     "dry_run": true,
     "results": [
       {"user_id": "...", "email": "...", "subject": "rendered subject", "body": "rendered body"},
       ...
     ]
   }
   ```

9. **Error handling:** If any individual send fails, log the error and continue with remaining recipients. Return per-recipient status. If ALL sends fail, return 500.
