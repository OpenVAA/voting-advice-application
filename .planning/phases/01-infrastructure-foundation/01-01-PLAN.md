<task type="auto">
  <name>Task 2: Rewrite playwright.config.ts with project dependencies and create strapiAdminClient.ts</name>
  <files>tests/playwright.config.ts, tests/tests/utils/strapiAdminClient.ts</files>
  <action>
1. Create `tests/tests/utils/strapiAdminClient.ts`:
   - Class `StrapiAdminClient` with constructor accepting optional `baseUrl` (defaults to `http://localhost:${process.env.STRAPI_PORT ?? '1337'}`).
   - `async login(email?, password?)` method: POST to `/admin/login` with `{ email, password }`, extract JWT from `response.data.token`, store internally. Default credentials: `mock.admin@openvaa.org` / `admin`.
   - Use Playwright's `request` API (`import { request } from '@playwright/test'`) to create an `APIRequestContext`.
   - `async importData(data)` method: POST to `/openvaa-admin-tools/import-data` with `data: JSON.stringify({ data })` and `Authorization: Bearer ${token}` header. CRITICAL: The Admin Tools controller does `JSON.parse(ctx.request.body)` so the body must be sent as a stringified JSON string via `data: JSON.stringify({ data })`, NOT as a pre-parsed object. Set `Content-Type: application/json`.
   - `async deleteData(data)` method: POST to `/openvaa-admin-tools/delete-data` with same pattern. `data` is `Record<string, string>` where values are externalId prefixes.
   - `async findData(collection, filters, populate?)` method: POST to `/openvaa-admin-tools/find-data`.
   - `async dispose()` method: disposes the APIRequestContext.
   - Export types: `ImportDataResult`, `DeleteDataResult`.
   - Include meaningful error messages on auth failure (403 vs 401) and request failures.

2. Rewrite `tests/playwright.config.ts`:

   - Remove `globalSetup` line entirely.
   - Remove the old `globalTimeout: 100000`.
   - Keep existing `dotenv.config()`, `TESTS_DIR`, `STORAGE_STATE` exports.
   - Keep `fullyParallel`, `forbidOnly`, `retries`, `workers`, `reporter`, `trace: 'on'` settings.
   - Keep `baseURL` logic using `FRONTEND_PORT`.
   - Remove global `storageState` from top-level `use` (it's now per-project).
   - Define 5 projects:
     a. `data-setup`: `testMatch: /data\.setup\.ts/`, `teardown: 'data-teardown'`
     b. `data-teardown`: `testMatch: /data\.teardown\.ts/`
     c. `auth-setup`: `testMatch: /auth\.setup\.ts/`, `dependencies: ['data-setup']`
     d. `candidate-app`: `testDir: './tests/specs/candidate'`, uses Desktop Chrome + `storageState: STORAGE_STATE`, `dependencies: ['auth-setup']`
     e. `voter-app`: `testDir: './tests/specs/voter'`, uses Desktop Chrome (no storageState), `dependencies: ['data-setup']`
   - Add a `timeout: 30000` per-test timeout (reasonable for E2E).

3. Create empty directories for the new structure:
