Update `apps/frontend/svelte.config.js` to enable runes mode globally. The final file must contain:

1. A `compilerOptions` block with `runes: true` at the config root level (for svelte-check and IDE support -- they cannot read dynamicCompileOptions)

2. A `vitePlugin` block with `dynamicCompileOptions` function that enables runes for all non-node_modules files (for Vite build/dev -- needed to exclude third-party Svelte libraries like svelte-visibility-change that use Svelte 3/4 syntax)

The exact config structure (per R6.1):

```javascript
import adapter from '@sveltejs/adapter-node';
import path from 'path';

/** @type {import('@sveltejs/kit').Config} */
const config = {
  compilerOptions: {
    runes: true
  },
  kit: {
    adapter: adapter({}),
    alias: {
      $types: path.resolve('./src/lib/types'),
      $voter: path.resolve('./src/lib/voter'),
      $candidate: path.resolve('./src/lib/candidate')
    },
    version: {
      pollInterval: 5 * 60 * 1000
    }
  },
  vitePlugin: {
    dynamicCompileOptions({ filename }) {
      if (!filename.includes('node_modules')) {
        return { runes: true };
      }
    }
  }
};

export default config;
```

CRITICAL placements (per research pitfall analysis):
- `dynamicCompileOptions` MUST be under `vitePlugin`, NOT at config root (root placement causes error: "Invalid options in svelte config")
- `compilerOptions.runes: true` MUST be at config root (for svelte-check compatibility)
- For node_modules files, the function returns void (undefined), which preserves legacy mode for third-party libs

After writing the config, clear Vite and Turborepo caches to prevent stale compilation:
```bash
rm -rf apps/frontend/node_modules/.vite .turbo/
```
