- id uuid PRIMARY KEY DEFAULT gen_random_uuid()
- account_id uuid NOT NULL REFERENCES accounts(id)
- name text NOT NULL
- default_locale text NOT NULL DEFAULT 'en'
- created_at timestamptz DEFAULT now()
- updated_at timestamptz DEFAULT now()

**Migration 00002_create_localization_functions.sql:**

Create `update_updated_at()` trigger function:
```sql
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

Create `get_localized(val jsonb, locale text, default_locale text)` function returning text, marked IMMUTABLE:
- Try val->>locale first
- Fall back to val->>default_locale
- Fall back to first available key via `(SELECT val ->> k FROM jsonb_object_keys(val) AS k LIMIT 1)`
- Return NULL if val is NULL

Apply updated_at triggers to accounts and projects tables.

Use Supabase timestamped migration naming. The `00001` and `00002` prefixes are placeholders -- use the actual Supabase timestamp format `YYYYMMDDHHMMSS`. Generate timestamps with current date, incrementing seconds (e.g., 20260312200000, 20260312200001). These must be sequential.

All content tables share a common column set derived from DataObjectData (mapped to snake_case):
```
id            uuid PRIMARY KEY DEFAULT gen_random_uuid()
project_id    uuid NOT NULL REFERENCES projects(id)
name          jsonb              -- localizable
short_name    jsonb              -- localizable
info          jsonb              -- localizable
color         text
color_dark    text
image         jsonb
sort_order    integer            -- NOT "order" (reserved word)
subtype       text
custom_data   jsonb
is_generated  boolean DEFAULT false
created_at    timestamptz DEFAULT now()
updated_at    timestamptz DEFAULT now()
```

Create the following tables with their entity-specific columns (all include the common set above plus project_id FK):

**elections:** election_date date, election_start_date date, election_type text, multiple_rounds boolean DEFAULT false, current_round integer DEFAULT 1

**constituency_groups:** (common columns only, no extra)

**constituencies:** keywords jsonb, parent_id uuid REFERENCES constituencies(id)

**constituency_group_constituencies:** (join table) constituency_group_id uuid REFERENCES constituency_groups(id) ON DELETE CASCADE, constituency_id uuid REFERENCES constituencies(id) ON DELETE CASCADE, PRIMARY KEY (constituency_group_id, constituency_id). No common columns -- just the FK pair.

**election_constituency_groups:** (join table) election_id uuid REFERENCES elections(id) ON DELETE CASCADE, constituency_group_id uuid REFERENCES constituency_groups(id) ON DELETE CASCADE, PRIMARY KEY (election_id, constituency_group_id). No common columns -- just the FK pair.

**organizations:** (common columns only -- answers added by Plan 02 answer alternative migrations)

**candidates:** first_name text NOT NULL, last_name text NOT NULL, organization_id uuid REFERENCES organizations(id). Note: auth_user_id added in Phase 10.

**factions:** (common columns only)

**alliances:** (common columns only)

**question_templates:** type text NOT NULL (one of the QuestionType values), settings jsonb, default_choices jsonb

**question_categories:** category_type text DEFAULT 'opinion', election_ids jsonb, election_rounds jsonb, constituency_ids jsonb, entity_type jsonb

**questions:** type text NOT NULL, category_id uuid NOT NULL REFERENCES question_categories(id), template_id uuid REFERENCES question_templates(id), choices jsonb, settings jsonb, election_ids jsonb, election_rounds jsonb, constituency_ids jsonb, entity_type jsonb, allow_open boolean DEFAULT true, required boolean DEFAULT true

**nominations:** entity_type text NOT NULL CHECK (entity_type IN ('candidate', 'organization', 'faction', 'alliance')), entity_id uuid NOT NULL, election_id uuid NOT NULL REFERENCES elections(id), constituency_id uuid NOT NULL REFERENCES constituencies(id), election_round integer DEFAULT 1, election_symbol text, parent_nomination_id uuid REFERENCES nominations(id), parent_entity_type text, unconfirmed boolean DEFAULT false. Note: entity_id cannot have a simple FK because it's polymorphic -- add a COMMENT explaining this and that trigger validation is deferred to Phase 10.

Apply `update_updated_at` trigger to EVERY table that has updated_at (all content tables, NOT join tables).

Add a CHECK constraint or comment on question_templates.type and questions.type to document the valid QuestionType enum values: 'text', 'number', 'boolean', 'image', 'date', 'multipleText', 'singleChoiceOrdinal', 'singleChoiceCategorical', 'multipleChoiceCategorical'.
