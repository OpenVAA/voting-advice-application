  <action>
Modify `tests/playwright.config.ts` to support visual and performance test isolation:

1. Add `snapshotPathTemplate` to the root config:
   ```
   snapshotPathTemplate: '{testDir}/__screenshots__/{testFileName}/{arg}{ext}'
   ```
   This creates a git-trackable directory structure under `tests/tests/__screenshots__/`.

2. Add default `expect.toHaveScreenshot` configuration:
   ```
   expect: {
     toHaveScreenshot: {
       threshold: 0.2,
       maxDiffPixelRatio: 0.01
     }
   }
   ```

3. Add `testIgnore` patterns to ALL existing projects that have a `testDir` pointing at spec subdirectories (`candidate-app`, `candidate-app-mutation`, `candidate-app-settings`, `voter-app`, `voter-app-settings`, `voter-app-popups`, all variant projects). For each, add `testIgnore` that excludes the visual and perf directories. Since these projects use `testDir: './tests/specs/voter'` (etc.), the visual/perf specs are already outside their testDir scope. However, the root-level projects (`data-setup`, `data-teardown`, `auth-setup`) use the root `testDir` which DOES include `specs/visual/` and `specs/perf/`. For these, ensure the `testMatch` patterns (e.g., `testMatch: /data\.setup\.ts/`) already prevent matching visual/perf files (they do, since they use specific file patterns).

   IMPORTANT: Verify that the existing `testIgnore: ['**/*.test.ts']` at the root level does NOT accidentally match the new spec files (it won't -- they use `.spec.ts`). No change needed at root `testIgnore`.

   The key insight: Since all existing projects either (a) use `testDir` pointing at `specs/candidate/`, `specs/voter/`, or `specs/variants/` subdirectories (which don't contain `visual/` or `perf/`), or (b) use explicit `testMatch` patterns that only match specific filenames, the new `specs/visual/` and `specs/perf/` directories are ALREADY excluded from all existing projects by the project-level `testDir`/`testMatch` scoping. No `testIgnore` modifications are needed on individual projects.

   BUT: Confirm this reasoning by checking if any project uses the root `testDir` without a `testMatch`. Currently, `data-setup`, `data-teardown`, and `auth-setup` all use `testMatch` patterns. No project relies on wildcard testDir matching. Therefore, no config changes are needed for exclusion from default runs.

4. Do NOT add `grepInvert` -- per the research, there's an open question about whether `--grep @visual` would override `grepInvert: [/@visual/]`. The directory-based isolation described above is sufficient and avoids this pitfall entirely.

Summary: The only config changes needed are `snapshotPathTemplate` and `expect.toHaveScreenshot` defaults. The directory structure inherently isolates visual/perf specs from default project runs.

  <name>Task 2: Create visual regression spec with 4 screenshot tests</name>
  <files>tests/tests/specs/visual/visual-regression.spec.ts</files>
  <action>
Create `tests/tests/specs/visual/visual-regression.spec.ts` with 4 screenshot tests covering 2 pages at 2 viewports.

Structure the file with 4 test.describe blocks, each tagged `@visual`:

**1. Voter Results - Desktop (1280x720)**
- Import `voterTest` from `../../fixtures/voter.fixture`
- Use `answeredVoterPage` fixture to land on results page
- `await page.waitForLoadState('networkidle')` for stable rendering
- `await page.getByTestId(testIds.voter.results.list).waitFor({ state: 'visible' })` to confirm results rendered
- `await expect(page).toHaveScreenshot('voter-results-desktop.png', { fullPage: true, animations: 'disabled' })`
- Note: Desktop viewport is already the default from `devices['Desktop Chrome']` in the project config. Since this spec runs outside any project, explicitly set viewport via `voterTest.use({ viewport: { width: 1280, height: 720 } })`.

**2. Voter Results - Mobile (Chromium-based mobile)**
- Same voterTest approach but with mobile viewport override
- Use custom Chromium-based mobile config: `voterTest.use({ viewport: { width: 390, height: 844 }, isMobile: true, hasTouch: true })` -- iPhone 14-like dimensions on Chromium (NOT `devices['iPhone 14']` which defaults to WebKit)
- Same fixture + waitForLoadState + screenshot
- `await expect(page).toHaveScreenshot('voter-results-mobile.png', { fullPage: true, animations: 'disabled' })`

**3. Candidate Preview - Desktop (1280x720)**
- Import `test, expect` from `../../fixtures` (standard fixture, not voterTest)
- Import `{ STORAGE_STATE }` from `../../../playwright.config`
- `test.use({ storageState: STORAGE_STATE, viewport: { width: 1280, height: 720 } })` for candidate auth
- Navigate to preview: `await page.goto(buildRoute({ route: 'CandAppPreview', locale: 'en' }))`
- Wait for preview container: `await page.getByTestId(testIds.candidate.preview.container).waitFor({ state: 'visible' })`
- `await page.waitForLoadState('networkidle')`
- `await expect(page).toHaveScreenshot('candidate-preview-desktop.png', { fullPage: true, animations: 'disabled' })`

**4. Candidate Preview - Mobile**
- Same as desktop but with mobile viewport override: `test.use({ storageState: STORAGE_STATE, viewport: { width: 390, height: 844 }, isMobile: true, hasTouch: true })`
- `await expect(page).toHaveScreenshot('candidate-preview-mobile.png', { fullPage: true, animations: 'disabled' })`
