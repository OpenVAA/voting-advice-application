**voter-bulk-read-jsonb.sql** (also serves as concurrent-read with `-c 100/500`):
```sql
\set project_num random(1, 5)
\set constituency_num random(1, 10)
-- Compute full constituency offset: (project_num-1)*10 + constituency_num
\set constituency_offset :project_num * 10 - 10 + :constituency_num
SELECT c.id, c.first_name, c.last_name, c.name, c.answers
FROM candidates c
INNER JOIN nominations n ON n.candidate_id = c.id
WHERE n.constituency_id = ('00000000-0000-0000-0002-' || lpad(:constituency_offset::text, 12, '0'))::uuid
  AND n.project_id = ('00000000-0000-0000-0001-' || lpad(:project_num::text, 12, '0'))::uuid
  AND c.published = true;
```

**voter-bulk-read-relational.sql**:
Same constituency selection as JSONB. Two approaches -- use the single-query JSON aggregation approach for fair comparison:
```sql
SELECT c.id, c.first_name, c.last_name, c.name,
  COALESCE(
    (SELECT jsonb_object_agg(a.question_id::text, jsonb_build_object('value', a.value))
     FROM answers a
     WHERE a.entity_id = c.id AND a.entity_type = 'candidate'),
    '{}'::jsonb
  ) AS answers
FROM candidates c
INNER JOIN nominations n ON n.candidate_id = c.id
WHERE n.constituency_id = :'constituency_uuid'
  AND n.project_id = :'project_uuid'
  AND c.published = true;
```
Use the same `\set` variable pattern as JSONB for random constituency/project selection.

**candidate-write-jsonb.sql** (single answer update via jsonb_set):
- Pick a random candidate from the generated data
- Pick a random question from the 50 per project
- Use `UPDATE candidates SET answers = jsonb_set(COALESCE(answers, '{}'::jsonb), ARRAY[question_uuid_str], jsonb_build_object('value', random_value)) WHERE id = candidate_uuid`
- Since candidates use gen_random_uuid(), we need a different approach: use a subquery to pick a random candidate ID. Or: select a random candidate via `(SELECT id FROM candidates WHERE project_id = :proj OFFSET floor(random() * :scale) LIMIT 1)`. But pgbench cannot do dynamic subquery assignment.
- Better approach: use `\set candidate_offset random(1, :scale)` and query with OFFSET/LIMIT to get a candidate ID, then update. But pgbench does not support multi-statement transactions well for this.
- Simplest approach that works: use a single UPDATE with a subquery for random candidate selection:
```sql
\set project_num random(1, 5)
\set question_num random(1, 50)
\set answer_value random(1, 5)
\set question_offset :project_num * 50 - 50 + :question_num
UPDATE candidates SET answers = jsonb_set(
  COALESCE(answers, '{}'::jsonb),
  ARRAY[('00000000-0000-0000-0003-' || lpad(:question_offset::text, 12, '0'))],
  jsonb_build_object('value', :answer_value)
)
WHERE id = (
  SELECT id FROM candidates
  WHERE project_id = ('00000000-0000-0000-0001-' || lpad(:project_num::text, 12, '0'))::uuid
  ORDER BY id OFFSET floor(random() * 100) LIMIT 1
);
```
Note: The OFFSET random approach is approximate but sufficient for benchmarking. Alternatively, use ctid-based random selection or just update all candidates round-robin.

**candidate-write-relational.sql** (single answer upsert):
```sql
\set project_num random(1, 5)
\set question_num random(1, 50)
\set answer_value random(1, 5)
\set question_offset :project_num * 50 - 50 + :question_num
INSERT INTO answers (project_id, entity_id, entity_type, question_id, value)
SELECT
  ('00000000-0000-0000-0001-' || lpad(:project_num::text, 12, '0'))::uuid,
  c.id,
  'candidate',
  ('00000000-0000-0000-0003-' || lpad(:question_offset::text, 12, '0'))::uuid,
  to_jsonb(:answer_value)
FROM candidates c
WHERE c.project_id = ('00000000-0000-0000-0001-' || lpad(:project_num::text, 12, '0'))::uuid
ORDER BY c.id OFFSET floor(random() * 100) LIMIT 1
ON CONFLICT (entity_id, question_id)
DO UPDATE SET value = EXCLUDED.value;
```

**candidate-full-save-relational.sql** (50 upserts in a transaction):
- Use a single multi-value INSERT...ON CONFLICT for all 50 questions for a random candidate
- Build with a SELECT from questions table:
```sql
\set project_num random(1, 5)
WITH target AS (
  SELECT id FROM candidates
  WHERE project_id = ('00000000-0000-0000-0001-' || lpad(:project_num::text, 12, '0'))::uuid
  ORDER BY id OFFSET floor(random() * 100) LIMIT 1
)
INSERT INTO answers (project_id, entity_id, entity_type, question_id, value)
SELECT
  ('00000000-0000-0000-0001-' || lpad(:project_num::text, 12, '0'))::uuid,
  target.id,
  'candidate',
  q.id,
  to_jsonb((floor(random() * 5) + 1)::int)
FROM questions q, target
WHERE q.project_id = ('00000000-0000-0000-0001-' || lpad(:project_num::text, 12, '0'))::uuid
ON CONFLICT (entity_id, question_id)
DO UPDATE SET value = EXCLUDED.value;
```

**aggregation-jsonb.sql** (GROUP BY answers per question):
```sql
\set project_num random(1, 5)
SELECT key AS question_id, count(*) AS answer_count,
  avg((value->>'value')::numeric) AS avg_value
FROM candidates c,
  jsonb_each(c.answers) AS kv(key, value)
WHERE c.project_id = ('00000000-0000-0000-0001-' || lpad(:project_num::text, 12, '0'))::uuid
  AND c.published = true
GROUP BY key;
```

**aggregation-relational.sql** (simple GROUP BY):
```sql
\set project_num random(1, 5)
SELECT question_id, count(*) AS answer_count,
  avg((value->>'value')::numeric) AS avg_value
FROM answers
WHERE project_id = ('00000000-0000-0000-0001-' || lpad(:project_num::text, 12, '0'))::uuid
  AND entity_type = 'candidate'
GROUP BY question_id;
```
