A recursive JSONB deep merge helper function:
```sql
CREATE OR REPLACE FUNCTION jsonb_recursive_merge(base jsonb, patch jsonb)
RETURNS jsonb
LANGUAGE sql
IMMUTABLE
AS $$
  SELECT CASE
    WHEN jsonb_typeof(base) = 'object' AND jsonb_typeof(patch) = 'object' THEN
      (SELECT jsonb_object_agg(
        COALESCE(k, pk),
        CASE
          WHEN k IS NOT NULL AND pk IS NOT NULL THEN jsonb_recursive_merge(base -> k, patch -> pk)
          WHEN pk IS NOT NULL THEN patch -> pk
          ELSE base -> k
        END
      )
      FROM (
        SELECT DISTINCT COALESCE(k, pk) AS key, k, pk
        FROM jsonb_object_keys(base) k
        FULL OUTER JOIN jsonb_object_keys(patch) pk ON k = pk
      ) keys)
    ELSE patch
  END;
$$;
```

Then the `merge_jsonb_column` RPC:
```sql
CREATE OR REPLACE FUNCTION merge_jsonb_column(
  p_table_name text,
  p_column_name text,
  p_row_id uuid,
  p_partial_data jsonb
)
RETURNS void
LANGUAGE plpgsql
SECURITY INVOKER
AS $$
BEGIN
  EXECUTE format(
    'UPDATE %I SET %I = jsonb_recursive_merge(%I, $1) WHERE id = $2',
    p_table_name, p_column_name, p_column_name
  ) USING p_partial_data, p_row_id;
END;
$$;
```

**Class `SupabaseAdminClient`:**
- Constructor takes optional `url`, `serviceRoleKey`, `projectId`. Creates `@supabase/supabase-js` client with `createClient()`, options: `{ auth: { autoRefreshToken: false, persistSession: false } }`. Stores `projectId` (default: `TEST_PROJECT_ID`).
- No `login()` or `dispose()` needed -- stateless via service_role key.

**Methods (each must match the usage patterns found in consumer files):**

1. `async bulkImport(data: Record<string, unknown[]>)`: Calls `this.client.rpc('bulk_import', { data })`. Throws on error. Returns the result JSON.

2. `async bulkDelete(collections: Record<string, { prefix?: string; ids?: string[]; external_ids?: string[] }>)`: Calls `this.client.rpc('bulk_delete', { data: { project_id: this.projectId, collections } })`. Throws on error. Returns result JSON.

3. `async importAnswers(data: Record<string, unknown[]>)`: After bulk_import inserts questions and candidates, this method resolves `answersByExternalId` on candidates. For each collection entry that has `answersByExternalId`:
   a. Collect all question external_ids referenced across all candidates
   b. Query `this.client.from('questions').select('id, external_id').in('external_id', [...questionExtIds]).eq('project_id', this.projectId)` to get UUID map
   c. For each candidate with `answersByExternalId`, build `answers` JSONB keyed by question UUID
   d. Look up candidate id by external_id, then `this.client.from('candidates').update({ answers }).eq('id', candidateId)`

4. `async linkJoinTables(data: Record<string, unknown[]>)`: After bulk_import, populate M:N join tables. Scans the imported data for:
   - `constituency_groups[].constituencies` -- each has `{ externalId: [...] }` pattern. Resolve constituency_group external_id to UUID, resolve each constituency external_id to UUID, insert into `constituency_group_constituencies`.
   - `elections[].constituencyGroups` or `elections[].constituency_groups` -- resolve election external_id to UUID, resolve each constituency_group external_id to UUID, insert into `election_constituency_groups`.
   Uses service_role client to insert directly into join tables.

5. `async updateAppSettings(partialSettings: Record<string, unknown>)`: Fetches app_settings row id for `this.projectId`, then calls `this.client.rpc('merge_jsonb_column', { p_table_name: 'app_settings', p_column_name: 'settings', p_row_id: row.id, p_partial_data: partialSettings })`. Throws on error.

6. `async findData(collection: string, filters: Record<string, unknown>)`: Translate Strapi-style `{ externalId: { $eq: 'value' } }` filters to PostgREST. Build a query `this.client.from(collection).select('*')`. For each filter key, convert camelCase to snake_case (e.g., `externalId` -> `external_id`), extract the value from `{ $eq: value }`, and chain `.eq(snakeKey, value)`. Add `.eq('project_id', this.projectId)`. Returns `{ type: 'success', data: rows }` for backward compat. The `documentId` field that Strapi used maps to `id` in Supabase, so alias `id` as `documentId` in results for backward compat during transition -- actually simpler: just add `documentId: row.id` to each result row.

7. `async query(collection: string)`: Returns `this.client.from(collection).select('*').eq('project_id', this.projectId)` for generic PostgREST queries. Used by specs that need more complex queries (e.g., startfromcg spec finding constituency_groups).

8. `async update(collection: string, id: string, data: Record<string, unknown>)`: Generic update: `this.client.from(collection).update(data).eq('id', id)`. Throws on error.

9. `async setPassword(email: string, password: string)`: Uses `this.client.auth.admin.listUsers()`, finds user by email, calls `this.client.auth.admin.updateUserById(user.id, { password })`. Throws if user not found or update fails.

10. `async forceRegister(candidateExternalId: string, email: string, password: string)`:
    a. `this.client.auth.admin.createUser({ email, password, email_confirm: true })`
    b. Look up candidate id: `this.client.from('candidates').select('id').eq('external_id', candidateExternalId).eq('project_id', this.projectId).single()`
    c. Insert user_role: `this.client.from('user_roles').insert({ user_id: user.id, role: 'candidate', scope_type: 'candidate', scope_id: candidate.id, project_id: this.projectId })`
    d. Link to candidate: `this.client.from('candidates').update({ auth_user_id: user.id }).eq('id', candidate.id)`

11. `async unregisterCandidate(email: string)`:
    a. `this.client.auth.admin.listUsers()`, find by email. If not found, return (already unregistered).
    b. Clear auth_user_id: `this.client.from('candidates').update({ auth_user_id: null }).eq('auth_user_id', user.id)`
    c. Delete user_roles: `this.client.from('user_roles').delete().eq('user_id', user.id)`
    d. Delete auth user: `this.client.auth.admin.deleteUser(user.id)`

12. `async sendEmail(params: { candidateExternalId: string; subject: string; content: string })`: Look up candidate by external_id to get their id, then call the invite-candidate Edge Function via `this.client.functions.invoke('invite-candidate', { body: { candidate_id: candidate.id, subject: params.subject, content: params.content } })`. Alternatively, for test purposes, use `this.client.auth.admin.inviteUserByEmail(email)` if the Edge Function is not available in test environment. The executor should check which approach works with `supabase start` and implement accordingly.

13. `async sendForgotPassword(email: string)`: Uses `this.client.auth.admin.generateLink({ type: 'recovery', email })`. This generates the recovery link without sending an email via external service. The link properties are returned directly. For E2E tests that need the link delivered via email, use `this.client.auth.resetPasswordForEmail(email, { redirectTo: ... })` which sends via Inbucket in local dev.

14. `async deleteAllTestUsers()`: Helper for teardown. Lists all users via `this.client.auth.admin.listUsers()`, filters to test emails (containing 'openvaa.org' or 'test'), deletes user_roles and auth users for each.
