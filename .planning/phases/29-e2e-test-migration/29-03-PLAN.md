<!-- Inbucket REST API v1 -->
Inbucket (at localhost:54324 via supabase start):
- GET /api/v1/mailbox/{name} -- list messages (name = email local part, e.g., "mock.candidate.2" from "mock.candidate.2@openvaa.org")
- GET /api/v1/mailbox/{name}/{id} -- get message with parsed body: { body: { text, html } }
- DELETE /api/v1/mailbox/{name} -- purge mailbox

  date: string;

}

**Interface for full message (with body):**
```typescript
interface InbucketMessage extends InbucketMessageHeader {
  body: { text: string; html: string };
}
```

**Functions to implement (preserving the same export names for consumer compatibility):**

1. `fetchEmails(recipientEmail: string): Promise<InbucketMessageHeader[]>`:
   - `const mailbox = getMailboxName(recipientEmail)`
   - `const response = await fetch(\`${INBUCKET_URL}/api/v1/mailbox/${mailbox}\`)`
   - If not ok, return empty array
   - Return `response.json()`

2. `getLatestEmailHtml(recipientEmail: string, skipCount = 0): Promise<string | undefined>`:
   - Fetch message list via `fetchEmails(recipientEmail)`
   - If `messages.length <= skipCount`, return undefined
   - Get the target message: `messages[messages.length - 1 - skipCount]` (latest minus skip)
   - Fetch full message: `fetch(\`${INBUCKET_URL}/api/v1/mailbox/${getMailboxName(recipientEmail)}/${target.id}\`)`
   - Parse as InbucketMessage, return `message.body?.html ?? undefined`

3. `extractLinkFromHtml(html: string): string | undefined`:
   - Unchanged: `const $ = load(html); return $('a').first().attr('href') ?? undefined;`

4. `getRegistrationLink(recipientEmail: string): Promise<string>`:
   - Same logic as before: call getLatestEmailHtml, extract link, throw descriptive errors

5. `countEmailsForRecipient(recipientEmail: string): Promise<number>`:
   - `const messages = await fetchEmails(recipientEmail); return messages.length;`

6. `purgeMailbox(recipientEmail: string): Promise<void>`:
   - NEW helper: `await fetch(\`${INBUCKET_URL}/api/v1/mailbox/${getMailboxName(recipientEmail)}\`, { method: 'DELETE' })`

**Key differences from old implementation:**
- No MIME parsing (Inbucket returns pre-parsed body.html)
- No Playwright request context (use native fetch)
- Mailbox-based filtering (no scanning all messages for recipient)
- The `skipCount` parameter in `getLatestEmailHtml` now works by indexing from the end of the mailbox-specific list (more reliable than the old approach of scanning all SES messages)
