<task id="39-01-02">
<title>Add supabase-tests (pgTAP) job</title>
<read_first>
- .github/workflows/main.yaml (current CI workflow after task 01)
</read_first>
<action>
Add `supabase-tests` job to `.github/workflows/main.yaml` after the `skill-drift-check` job:
```yaml
supabase-tests:
  runs-on: ubuntu-latest
  steps:
    - name: "Checkout source code"
      uses: actions/checkout@v4

    - uses: dorny/paths-filter@v3
      id: changes
      with:
        filters: |
          supabase:
            - 'apps/supabase/**'
            - 'packages/supabase-types/**'

    - uses: supabase/setup-cli@v1
      if: steps.changes.outputs.supabase == 'true'
      with:
        version: latest

    - name: "Start Supabase"
      if: steps.changes.outputs.supabase == 'true'
      working-directory: apps/supabase
      run: supabase start

    - name: "Run pgTAP tests"
      if: steps.changes.outputs.supabase == 'true'
      working-directory: apps/supabase
      run: supabase test db

    - name: "Stop Supabase"
      if: always() && steps.changes.outputs.supabase == 'true'
      working-directory: apps/supabase
      run: supabase stop
```
Key details:
- Uses `dorny/paths-filter@v3` to only run when `apps/supabase/**` or `packages/supabase-types/**` files change
- Uses `supabase/setup-cli@v1` with `version: latest`
- Runs from `apps/supabase` working directory
- Always stops Supabase (cleanup)
</action>
<acceptance_criteria>
- `.github/workflows/main.yaml` contains `supabase-tests:` job
- Job uses `dorny/paths-filter@v3` with supabase filter paths
- Job uses `supabase/setup-cli@v1`
- Job has `working-directory: apps/supabase` for start/test/stop steps
- Job runs `supabase test db` command
- Stop step has `if: always()` condition
</acceptance_criteria>
</task>
