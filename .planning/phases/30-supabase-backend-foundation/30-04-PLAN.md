<task id="30-04-01">
<title>Verify supabase start and migrations</title>

<action>
1. Ensure Docker is running
2. Run from repository root:
   ```bash
   cd apps/supabase
   npx supabase start
   ```
3. Verify the output shows:
   - All services are healthy (API, DB, Studio, Inbucket, Edge Runtime)
   - Database is running on port 54322
   - API is running on port 54321

   ```bash
   npx supabase db reset  # Clean reset to verify migrations + seed
   ```

- config.toml project_id is set to "openvaa-local"
</action>

- `cd apps/supabase && npx supabase status | grep "DB URL"` shows port 54322

</acceptance_criteria>

<task id="30-04-03">
<title>Verify Edge Functions are deployable and respond</title>
<read_first>
- apps/supabase/supabase/functions/invite-candidate/index.ts
- apps/supabase/supabase/functions/send-email/index.ts
- apps/supabase/supabase/functions/signicat-callback/index.ts
- apps/supabase/supabase/config.toml (edge_runtime config)
</read_first>
<action>
With the local Supabase instance running:

1. Edge Functions are auto-served by `supabase start` when `edge_runtime.enabled = true` (confirmed in config.toml).

2. Verify each function responds:
   ```bash
   # invite-candidate — should return 400 (missing body) or 405 (GET not allowed)
   curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:54321/functions/v1/invite-candidate

   # send-email — should return 400 (missing body) or 405 (GET not allowed)
   curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:54321/functions/v1/send-email

   # signicat-callback — should return 400 (missing body) or 405 (GET not allowed)
   curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:54321/functions/v1/signicat-callback
   ```

3. Expected: Each function returns a non-500 HTTP status code (405 for GET requests since they only accept POST, or similar). A 500 would indicate the function failed to load/parse.

4. Verify with POST (expect 400 or 401 since we're not providing valid auth/body):
   ```bash
   curl -s -X POST http://127.0.0.1:54321/functions/v1/invite-candidate \
     -H "Content-Type: application/json" \
     -d '{}' | head -1
   ```
   Expected: JSON response with an error message (not a crash/500).

Note: The CONTEXT.md says Edge Functions are "invite-candidate, send-email, signicat-callback" but the roadmap says "preregister, send-email, admin". The actual function names from the branch are invite-candidate, send-email, and signicat-callback. Use the actual names.
</action>
<acceptance_criteria>
- `curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:54321/functions/v1/invite-candidate` returns a non-500 status code (405 or similar)
- `curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:54321/functions/v1/send-email` returns a non-500 status code
- `curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:54321/functions/v1/signicat-callback` returns a non-500 status code
- `curl -s -X POST http://127.0.0.1:54321/functions/v1/invite-candidate -H "Content-Type: application/json" -d '{}'` returns a JSON response (not a crash)
</acceptance_criteria>
</task>

